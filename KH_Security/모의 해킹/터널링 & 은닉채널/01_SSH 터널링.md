# SSH 터널링

## 개요
- SSH 터널링은 SSH 연결을 이용하여 다른 네트워크 트래픽을 안전하게 우회 전달하는 방식입니다.
- 교안에서는 SSH 서버를 경유하여 원래 직접 접근하기 어려운 웹 자원에 접속하는 흐름으로 설명합니다.

---

## 실습 환경 흐름
- 클라이언트가 SSH 서버에 먼저 접속한 뒤, 그 SSH 연결을 이용해 프록시 통로를 만들고,  
  브라우저가 그 프록시를 통해 웹 서버로 나가도록 구성하는 방식입니다.
- 즉, 브라우저가 웹 서버에 직접 가는 것이 아니라, 로컬에 만들어진 프록시 포트로 보내고,  
  그 요청이 SSH 서버를 거쳐 목적지로 전달됩니다.

- **프록시** : 직접 통신하지 않고 다른 서버를 통해서 통신을 하는 방식입니다.

### SSH 접속 설정

- PuTTY에서 SSH 서버 접속 정보를 설정합니다.

![01](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/01.png)

- Connection -> SSH -> Tunnels 클릭을 하여 다음과 같이 입력 후 ADD 버튼을 누릅니다.

![02](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/02.png)

![03](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/03.png)

- Add 버튼을 누른 후 위의 결과처럼 `D4444`라는 Source port를 설정합니다.
- 4444로 설정하는 것은 브라우저가 외부로 나갈 때 직접 나가는 것이 아니라,  
  `localhost:4444`에 먼저 보내도록 만드는 것입니다.

### Windows Proxy 설정

- `제어판` -> `네트워크 및 인터넷` -> `인터넷 옵션` -> `연결` -> `LAN 설정`을 클릭합니다.
- 다음과 같은 화면에서 고급 버튼을 눌러줍니다.

![04](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/04.png)

- 그 이후에 다음과 같이 설정해줍니다.

![05](/KH_Security/모의%20해킹/터널링%20%26%20은닉채널/img/05.png)

- SOCKS 프록시 사용
  - 주소: `localhost`
  - 포트: `4444`

- 의미 :
  - 로컬 PC(127.0.0.1)에 SOCKS 프록시 서버가 실행
  - 해당 포트를 통해 트래픽이 전달됨
