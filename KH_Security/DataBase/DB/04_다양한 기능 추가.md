# 다양한 기능 추가 (삭제, 검색, 화면단위 출력)

## 개요

- 출력 프로그램에 단순 조회만 하는 기능에서 한 단계 더 나아가  
DB 내용을 보여주는 프로그램을 사용자가 직접 조작할 수 있는 프로그램으로 확장하는 단계라고 보면 됩니다.

---

## 삭제 기능

- 삭제 기능 코드
```php
  $del=$_GET['del']; 
  
  if (empty($del)){}    
  else {                             
    $sql="delete from student where sno='$del'"; 
    $result=oci_parse($conn,$sql); // SQL문을 DB가 이해할 수 있도록 준비하는 단계 (컴파일 느낌)

    oci_execute($result); // 실제로 DELETE 쿼리를 DB에 실행
    oci_free_statement($result); // 사용한 SQL 실행 객체($result)를 메모리에서 해제
  }
```

### 결과 화면 및 동작방식

- 동작 방식 

## 검색 기능

- 검색 기능 코드
```php
  $search = isset($_GET['search']) ? $_GET['search'] : '';

  if (empty($search)) {
      $sql = "select sno,sname,sex,syear,major,to_char(avr,'0.00') avr
              from student order by sno";
  }
  else {
      $sql = "select sno,sname,sex,syear,major,to_char(avr,'0.00') avr
              from student
              where sname like '%{$search}%' order by sno";
  }

  <form name="search" method="get" action="st_vi.php">
    검색창 &nbsp;<input type="text" name="search">
    <input type="submit" name="확인" value="확인">
  </form>
```

### 결과 화면 및 동작방식

![07](/KH_Security/DataBase/img/07.png)

- 동작 방식
```text
  - 사용자가 검색창에 값을 입력하고 "확인" 버튼을 누릅니다.
  - form의 method="get" 방식으로 st_vi.php?search=값 형태로 전달됩니다.
  - PHP에서 $_GET['search'] 값을 받습니다.
  
  - $search 값이 비어있는지(empty) 확인합니다.
  - 값이 없으면 전체 조회 SQL을 만듭니다.
    - select ... from student order by snㅐ
  
  - 값이 있으면 검색 SQL을 만듭니다.
    - where sname like '%검색값%'
  
  - 생성된 SQL을 실행한 뒤 결과를 화면에 출력합니다.
```

- 즉, 사용자가 검색값을 입력하면  
그 값을 기준으로 SQL 조건을 바꿔서 원하는 데이터만 조회하는 구조입니다.

## 화면 단위 출력

- 화면 단위 출력코드
```php
  $scale=5; // 화면에 출력할 행의 개수

  $start = $_GET['start'];

  if (empty($start)){ $start=0;} // 첫 화면은 0에서 시작, 배열의 키값은 0에서 시작

  echo("<table border='0'>");
  for ($i=$start; $i<($start+$scale); $i++) {
    if($i<$row_num){
      echo("
       <tr>
         <td width='50'><p align='center'>{$row['SNO'][$i]}</p></td>
         <td width='80'><p align='center'>{$row['SNAME'][$i]}</p></td>
         <td width='20'><p align='center'>{$row['SEX'][$i]}</p></td>
         <td width='20'><p align='center'>{$row['SYEAR'][$i]}</p></td>
         <td width='50'><p align='center'>{$row['MAJOR'][$i]}</p></td>
         <td width='30'><p align='center'>{$row['AVR'][$i]}</p></td>
        </tr>
      ");
    }
  }
  echo("</table><hr>");

  $p=$start-$scale; // 이전 화면의 시작 위치
  $n=$start+$scale; // 다음 화면의 시작 위치

  if($p>=0)
    echo("<a href=./st_vi.php?start=$p>[이전페이지]</a>&nbsp;&nbsp;");
  else
    echo("이전페이지&nbsp;&nbsp;");

  if($n<$row_num)
    echo("<a href=./st_vi.php?start=$n>[다음페이지]</a>");
  else
    echo("다음페이지");
```

### 결과화면 및 동작방식

![08](/KH_Security/DataBase/img/08.png)

- 동작 방식
```text
- 한 화면에 출력할 개수를 $scale=5 로 설정합니다.
- $_GET['start'] 값을 받아 현재 페이지 시작 위치를 결정합니다.
- start 값이 없으면 0으로 설정합니다.  
첫 페이지는 배열의 0번 인덱스부터 시작
- for문을 이용해서 start부터 (start + scale)까지 반복합니다.
- 반복하면서 현재 인덱스 i가 전체 행 개수($row_num)보다 작은 경우만 출력합니다.  
그리고 해당 범위의 데이터만 테이블로 출력됩니다.
```

- 즉, 전체 데이터를 다 보여주는 것이 아니라  
현재 페이지에 해당하는 일부 데이터만 출력합니다.

## 검색 & 화면 단위 출력

- 검색했을 때 검색 결과를 유지한 채로 페이지 이동하도록 구현하는 코드입니다.
- 기존 페이지 처리에서 search 값을 함께 전달하면 됩니다.
- GET 파라미터로 search 값을 유지하면서 start 값과 함께 넘깁니다.

```php
  $search = $_GET['search'];
  if (empty($search)) // $search 변수가 없을 때 $sql
  {
      $sql="select sno,sname,sex,major,syear,to_char(avr,'0.00') avr
            from student order by sno";
  }   
  
  else  // $search 변수가 있을 때 $sql
  { 
    $sql="select sno,sname,sex,major,syear,to_char(avr,'0.00') avr 
          from student 
          where sname like '%{$search}%' order by sno";
  }

  $result = oci_parse($conn, $sql);
  oci_execute($result);
  
  $row_num = oci_fetch_all($result, $row);
  oci_free_statement($result);
  oci_close($conn);
  
  $scale=5;                           // 화면에 출력할 행의 개수
  $start = $_GET['start'];       
  if (empty($start)){ $start=0;}    // 첫 화면은 0에서 시작, 배열의 키값은 0에서 시작

  echo("<table border='0'>");
  for ($i = $start; $i < ($start+$scale); $i++) {
    if($i < $row_num){      
      echo("
            <tr>
               <td width='50'><p align='center'>{$row['SNO'][$i]}</p></td> 
               <td width='80'><p align='center'>{$row['SNAME'][$i]}</p></td>
               <td width='20'><p align='center'>{$row['SEX'][$i]}</p></td>
               <td width='20'><p align='center'>{$row['SYEAR'][$i]}</p></td>
               <td width='50'><p align='center'>{$row['MAJOR'][$i]}</p></td> 
               <td width='30'><p align='center'>{$row['AVR'][$i]}</p></td>
            </tr>
      ");
    }
  }
  echo("</table><hr>");

  $p=$start-$scale;     // 이전 화면의 시작 위치
  $n=$start+$scale;     // 다음 화면의 시작 위치
  
  if($p >= 0)
    echo("<a href=./st_vi.php?start=$p&search=$search>[이전페이지]</a>&nbsp;&nbsp;");
  else
    echo("이전페이지&nbsp;&nbsp;");
  
  if($n < $row_num)
    echo("<a href=./st_vi.php?start=$n&search=$search>[다음페이지]</a>");
  else
    echo("다음페이지");
?>

<form name='search' method='get' action='st_vi.php'>
  검색창 &nbsp;<input type='text' name='search'>

</form>
```

### 결과 화면 및 동작 방식

![09](/KH_Security/DataBase/img/09.png)

![10](/KH_Security/DataBase/img/10.png)

- 동작 방식
```
1. 사용자가 검색창에 이름 입력
2. form이 GET 방식으로 st_vi.php에 search 값을 전송
3. PHP가 $_GET['search']로 검색어를 받음
4. search 값이 있으면 WHERE sname LIKE '%검색어%' 조건으로 조회
5. 조회 결과를 5개씩 화면에 출력
6. 다음페이지 링크 생성 시 search 값도 같이 붙임
7. 다음페이지 클릭 시 URL에 search 값이 그대로 전달됨
8. 다음 페이지에서도 같은 검색 조건으로 다시 조회
```
