# 제약조건 정리 (PK, FK, UK, NOT NULL, CHECK)

## 제약조건이란
- 테이블 단위에서 정의되고 적용된다.
- 데이터가 삽입, 수정, 삭제될 때마다 규칙이 적용된다.
- 종속성이 존재하는 경우 테이블 삭제를 막아준다.
- 활성화/비활성화가 가능하다.
- 제약조건은 개체처럼 관리되므로 반드시 이름이 필요하다.

## 오라클에서 제공하는 제약조건
- PRIMARY KEY
- FOREIGN KEY
- UNIQUE KEY
- NOT NULL
- CHECK

---

## PRIMARY KEY (PK)

### 개념
- Primary Key, 주키, 주식별자
- 테이블마다 한 개만 정의 가능
- 테이블 내 모든 행을 유일하게 식별하는 컬럼

### 특징
- 중복 불가
- NULL 허용 안 함
- 고유 인덱스 자동 생성
- 모든 컬럼은 PK 컬럼에 함수적 종속 관계를 가진다
  - 예: 학번 → (이름, 학년, 학과)

---

## FOREIGN KEY (FK)

### 개념
- Foreign Key, 외부키
- 테이블 간 관계를 의미한다
- 항상 부모-자식 관계이다

### 특징
- 자식 테이블의 참조 컬럼에 지정한다
- 두 컬럼의 데이터 타입이 일치해야 한다
- PK 또는 UK만 참조 가능
- ON DELETE CASCADE 옵션 가능

---

## PK, FK 설정 방법

### 컬럼 레벨 설정

```sql
  CREATE TABLE 테이블 (
    컬럼 데이터타입 CONSTRAINT 제약조건 PRIMARY KEY
  );
  
  CREATE TABLE 테이블 (
    컬럼 데이터타입,
    CONSTRAINT 제약조건 FOREIGN KEY (컬럼)
    REFERENCES 참조테이블 (참조컬럼)
  );
```

### 테이블 레벨 설정

```sql
  CREATE TABLE 테이블 (
    컬럼 데이터타입,
    CONSTRAINT 제약조건 PRIMARY KEY (컬럼)
  );

  CREATE TABLE 테이블 (
    컬럼 데이터타입,
    CONSTRAINT 제약조건 FOREIGN KEY (컬럼)
    REFERENCES 참조테이블 (참조컬럼)
    ON DELETE CASCADE
 );
```

## 제약조건 검색

### 제약조건 조회

```sql
  SELECT c.table_name, c.constraint_name, c.constraint_type, c.status, s.column_name
  FROM user_constraints c, user_cons_columns s
  WHERE c.constraint_name = s.constraint_name
  AND c.table_name IN (검색대상테이블)
  ORDER BY c.table_name;
```

### 참조 관계 조회

```sql
  SELECT p.table_name 상위테이블, p.constraint_name 상위제약조건, c.table_name 하위테이블, c.constraint_name 참조제약조건
  FROM user_constraints p, user_constraints c
  WHERE c.r_constraint_name = p.constraint_name
  AND p.table_name IN (검색대상테이블)
  ORDER BY p.table_name;
```

## 제약조건에 의한 무결성 통제

- 제약조건은 입력, 수정, 삭제 작업 시 검사된다.
- 무결성 위반 시 오류 발생

### 예시 오류
- PK 중복 → ORA-00001
- 존재하지 않는 부모 참조 → ORA-02291
- 부모 삭제 시 자식 존재 → 삭제 불가

## 실습 과제 – 공장, 제품, 출고 상품 테이블
```sql
  CREATE TABLE factory (
    fno   VARCHAR2(4),
    fname VARCHAR2(40),
    loc   VARCHAR2(40),
    CONSTRAINT pk_factory PRIMARY KEY (fno)
  );
```

```sql
  CREATE TABLE goods (
    gno     VARCHAR2(6),
    gname   VARCHAR2(60),
    pri     NUMBER(10,2),
    fac_no  VARCHAR2(4),
    CONSTRAINT pk_goods PRIMARY KEY (gno),
    CONSTRAINT fk_goods_factory FOREIGN KEY (fac_no)
      REFERENCES factory(fno)
  );
```

```sql
  CREATE TABLE prod (
    s_num NUMBER,
    gno   VARCHAR2(6),
    pri   NUMBER(10,2),
    pdate DATE,
    CONSTRAINT pk_prod PRIMARY KEY (s_num),
    CONSTRAINT fk_prod_goods FOREIGN KEY (gno)
      REFERENCES goods(gno)
  );
```

## UK (Unique Key, 고유키)

- 특정 컬럼의 값이 중복되지 않도록 보장합니다.
- NULL은 허용됩니다.
```sql
  CREATE TABLE emp3 (
    eno VARCHAR2(4),
    gno VARCHAR2(13),
    CONSTRAINT emp4_gno_uk UNIQUE (gno)
  );
```
- 주민번호는 중복되면 안 되지만 NULL은 허용할 수 있습니다.

---

## NOT NULL 제약조건

- 해당 컬럼에 NULL 값을 허용하지 않습니다.
- 컬럼 레벨에서만 정의 가능합니다.
```sql
  CREATE TABLE emp3 (
    ename VARCHAR2(50) CONSTRAINT emp4_nu_ename NOT NULL
  );
```
- ALTER TABLE ... MODIFY로 추가합니다.

---

## CHECK 제약조건

- 입력값 조건을 지정하여 데이터 유효성을 검사합니다.
```sql
  CREATE TABLE emp3 (
    sex VARCHAR2(2),
    CONSTRAINT emp4_sex_ch CHECK (sex IN ('남','여'))
  );
  ```
- 성별 컬럼은 반드시 '남' 또는 '여'만 입력 가능합니다.

---

## 제약조건 추가
```sql
  ALTER TABLE class
  ADD CONSTRAINT class_cno_pk PRIMARY KEY (cno);
  
  ALTER TABLE st
  ADD CONSTRAINT st_cno_fk FOREIGN KEY (cno)
  REFERENCES class (cno);
```
---

## 제약조건 삭제
```sql
  ALTER TABLE st DROP CONSTRAINT st_cno_fk;
  ALTER TABLE class DROP CONSTRAINT class_cno_pk;
```
---

## 제약조건 비활성화 / 활성화
```sql
  ALTER TABLE st DISABLE CONSTRAINT st_sno_pk;
  ALTER TABLE st ENABLE CONSTRAINT st_sno_pk;
```
- DISABLE 시 관련 인덱스도 함께 삭제됩니다.
- PK 또는 UK를 비활성화하려면 참조 중인 FK부터 비활성화해야 합니다.

## 제약조건 적용 예시

```sql
  CREATE TABLE emp3 (
    eno VARCHAR2(4),
    ename VARCHAR2(50) CONSTRAINT emp3_ename_nu NOT NULL,
    gno VARCHAR2(13),
    sex VARCHAR2(2),
    CONSTRAINT emp3_eno_pk PRIMARY KEY (eno),
    CONSTRAINT emp3_gno_uk UNIQUE (gno),
    CONSTRAINT emp3_gno_ch CHECK (LENGTH(gno)=13),
    CONSTRAINT emp3_sex_ch CHECK (sex IN ('여','남'))
  );
```
