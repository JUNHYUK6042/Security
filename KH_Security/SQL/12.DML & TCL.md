# DML (INSERT, UPDATE, DELETE) & TCL

## DML과 TCL

- `DML (Data Manipulation Language)`
  - `INSERT` : 행 삽
  - `UPDATE` : 행 수정
  - `DELETE` : 행 삭제

- `TCL (Transaction Control Language)`
  - `COMMIT` : 작업 확정
  - `ROLLBACK` : 작업 취소

- **DML 문은 반드시 `COMMIT` 또는 `ROLLBACK`으로 종료해야 합니다.**

---

## INSERT 

- 형식
```sql
  INSERT INTO 테이블 [(컬럼, 컬럼, ...)]
  VALUES (값, 값, ...);
```

### INSERT (날짜 입력)

```sql
  INSERT INTO emp (eno, ename, hdate)
  VALUES ('0201', '안영숙', TO_DATE('2021/09/25:03:07:15', 'YYYY/MM/DD:HH24:MI:SS'));
```
‐ 날짜 입력에는 터미널의 날짜 형식에 따라 입력하는 것이 아니라 반드시  
TO_DATE 함수를 이용해서 날짜를 입력한다.

---

## UPDATE & DELETE

### UPDATE 형식
```sql
  UPDATE 테이블
  SET 컬럼 = 값, 컬럼 = 값, ...
  [WHERE 조건];
```

### DELETE 형식
```sql
  DELETE FROM 테이블
  [WHERE 조건];
```

#### UPDATE 검색 예시

- 김주란의 부서 번호를 10번으로, 급여는 10% 인상 검색
```sql
  UPDATE emp SET dno = '10', sal = sal*1.1
  WHERE ename = '김주란';

  COMMIT;
```

#### DELETE 검색 예시

- 10번 부서 사원의 정보를 삭제 검색
```sql
  DELETE FROM emp
  WHERE dno = '10';

  COMMIT;
```

---

## TCL (COMMIT, ROLLBACK)

```sql
  COMMIT : 영구 저장
  ROLLBACK : 마지막 커밋 시점으로 복구
```
