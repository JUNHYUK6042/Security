 # Index 생성 및 구조와 이해

## 인덱스란

- 인덱스는 테이블의 데이터를 빠르게 검색하기 위해 사용하는 구조입니다.
- 책의 "목차"와 같은 역할을 합니다.
- 인덱스에는 특정 컬럼 값과 해당 데이터의 위치(ROWID)가 저장됩니다.

---

## 인덱스 종류

### 고유성 기준
- Unique Index (고유 인덱스)
- Non Unique Index (비고유 인덱스)

### 물리 구조 기준
- B-Tree Index
- Bitmap Index

---

## Index 생성 방식

### 자동 생성
- `다음 제약조건이 설정되면 자동으로 인덱스가 생성된다.`

- PRIMARY KEY
- UNIQUE KEY

- **특징**
  - Unique Index가 자동 생성된다.

### 수동 생성
- `사용자가 직접 인덱스를 생성할 수도 있다.`

- **특징**
  - CREATE INDEX 명령 사용
  - Non Unique Index 생성
  - FK 컬럼에는 반드시 생성하는 것이 좋다.
