# KEY 관리

## 개요

- 키 관리는 암호 시스템에서 가장 중요한 요소로, 키의 생성부터 폐기까지 전체 생명주기를 관리하는 과정입니다.
- 암호 알고리즘이 안전해도 키 관리가 부실하면 전체 시스템은 쉽게 무너지게 됩니다..

---

## 세션 키 (Session Key)
- 통신할 때 사용하는 **일회성 비밀키**입니다.
- 통신이 끝나면 바로 폐기합니다.

### 특징
- 짧은 시간만 사용하기 때문에 보안성이 올라갑니다.
- 매 통신마다 새로 생성해서 공격이 어렵습니다.

---

### 물리적 전달

- 링크 암호와에서 적용 가능합니다.
- 단대단 암호화에서는 적용이 어렵습니다.

---

### 이전 키 사용하여 새로운 키 전송

- 링크 암호화와 단대단 암호화 모두 적용 가능
- 공격자가 키 하나만 알아도 다른 모든 키가 노출

---

### 제 3자를 이용한 키 분배 (KDC)

- 단대단 암호화에서 널리 사용됩니다.
- 사용자와 KDC는 각각 마스터키를 공유합니다.

---

## KDC를 이용한 비밀키 분배

### KDC를 이용한 키 분배 방법 (Kerboros)

- 키 교환 시스템이며, 대칭키를 이용하는 시스템입니다.
- 키 관리는 KDC에서 대신 관리해줍니다.
- 1차 도메인은 KDC, 2차 도메인은 서버이며, 실제 접속은 2차 도메인을 가는 것입니다.
- 클라이언트와 서버가 안전하게 통신을 하기 위해서 세션 키를 관리 해주는 것이 KDC(1차 도메인)입니다.

![01]()

- **과정**
```
1. A가 B와 통신하기 위해서 KDC에 B랑 통신할 수 있는 세션 키를 요청합니다. (ID[A] : A의 신용 정보)
2. KDC는 세션 키(ID[B])를 A에게 전달하며, 세션 키(ID[A])를 B에게 전달합니다. (ID[B] : B의 신용 정보)
  - A에게 가는 T는 미리 약속된 것을 의미하며, B에게 가는 T는 A의 신용정보입니다.
3. 서버에게 자신의 신용정보를 세션 키를 통해 보냅니다.
4. 그 이후에 서버는 클라이언트에게 신원 정보ID(B)와 확인하기 위한 T+1의 값을 보내줌으로써 서로 신원정보를 확인하고 통신합니다.
```

### 특징

- KDC가 중단되면 전체 통신이 불가능하기 때문에, 다중화가 필요합니다.
