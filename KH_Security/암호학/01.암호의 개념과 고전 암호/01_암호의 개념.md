# 암호의 개념과 고전 암호

## 개요

- 암호는 개방된 통신망에서 데이터를 안전하게 보호하기 위해 사용하는 기술로, 네트워크 환경에서 정보 유출을 방지하기 위한 핵심 요소입니다.

## 암호를 사용하는 이유

- 네트워크는 개방되어 있어 공격에 노출되며, 평문 상태로 전송 시 정보 탈취가 가능합니다.

---

## 암호 용어

![01](/KH_Security/암호학/01.암호의%20개념과%20고전%20암호/img/01.png)

- `평문` : 암호화 되기 전 보호되지 않은 원본 데이터입니다.  
- `암호 알고리즘(cryptographic)` : 암호를 만들어내는 시스템, 암호화키를 사용하여 평문을 암호화합니다.
- `암호화 키` : 암호문을 만들 때 암호 알고리즘에 사용되는 키입니다.
- `암호문` : 암호화 키와 암호 알고리즘을 통해 만들어낸 알 수 없는 데이터입니다.
- `암호해독` : 공격자가 암호문을 분석해서 원문을 알아내려는 시도하는 것입니다.
- `복호 알고리즘(decipher)` : 암호를 풀어 평문을 해석합니다.
- `복호화 키` : 암호화 된 데이터를 평문으로 하기 위해 복호 알고리즘에 사용되는 키입니다.

## 암호 원리

### 평문을 암호화하기 위한 연산자의 유형

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 치환 (Substitution) | 평문의 각 요소를 다른 값으로 변경 | 값 자체를 바꿈 | A → D (시저 암호) |
| 전치 (Transposition) | 평문의 요소 순서를 재배열 | 위치만 바뀜 | HELLO → LEOHL |
| 혼합 (Confusion) | 평문과 키의 관계를 복잡하게 만듦 | 키 추측 어렵게 함 | 특정 비트 선택 후 치환 |
| 확산 (Diffusion) | 평문의 변화가 암호문 전체에 퍼지도록 함 | 일부 변화 → 전체 영향 | 1비트 변경 시 전체 값 변화 |

### 사용된 키의 수

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 대칭키 (Symmetric Key) | 송신자와 수신자가 같은 키 사용 | 속도 빠름 / 키 관리 어려움 | AES, DES |
| 공개키 (Asymmetric Key) | 송신자와 수신자가 서로 다른 키 사용 | 보안성 높음 / 속도 느림 | RSA |

### 평문 처리 방식

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 블록 암호화 (Block Cipher) | 데이터를 일정 크기로 나눠 처리 | 정해진 크기 단위 처리 | AES (128bit 블록) |
| 스트림 암호화 (Stream Cipher) | 데이터를 연속적으로 처리 | 실시간 처리 / 빠름 | RC4 |

---

### 단대단 암호 (End-to-End Encryption)

- 송신자와 수신자 **종단 간에서만 암호화/복호화 수행**합니다.
- 중간 장비(라우터, 스위치)는 내용을 알 수 없습니다.
- 데이터는 전송 과정 내내 암호화 상태 유지합니다.

- 특징
  - 보안성 매우 높음
  - 중간 공격자가 내용 확인 불가능
  - 현대 인터넷 통신의 기본 방식

- 예시
  - HTTPS
  - 메신저 암호화 (카카오톡, WhatsApp 등)

### 링크 암호 (Link Encryption)

- 네트워크 **장비(라우터, 스위치 등) 단위로 암호화 수행**합니다.
- 각 구간(링크)마다 암호화/복호화 반복합니다.

- 특징
  - 구간별로 보호합니다.
  - 중간 장비에서는 평문으로 노출됩니다.
  - 장비 자체가 암호화 수행합니다.

- 예시
  - 군 통신망
  - 특정 전용망 (폐쇄망)

#### 단대단 암호나 링크 암호는 다음과 같은 이유들로 인해 가치가 없는 암호인 이유

- 링크 암호
  - 중간 장비에서 평문 노출됨
  - 내부 공격에 취약

- 단대단 암호 (과거 기준)
  - 키 관리 어려움
  - 구현 복잡

---

### 관용 암호
- 암호화와 복호화에 **같은 키** 사용합니다.

#### 특징
- 같은 알고리즘 + 같은 키 사용
- 송신자/수신자 둘 다 키를 알고 있어야 함
- 키는 반드시 비밀 유지
- 키 분배가 어려움 (핵심 문제)
- 디지털 서명 불가능
- 속도가 빠름

#### 구조 (시각화)
```
평문 → [같은 키] → 암호문 → [같은 키] → 평문
```

### 공개키 암호 (비대칭키, Asymmetric Key)
- 암호화와 복호화에 **서로 다른 키** 사용합니다.

#### 특징
- 공개키 + 개인키 사용
- 둘 중 하나만 알아도 암호화 가능
- 개인키는 반드시 비밀 유지
- 공개키는 누구에게나 공개 가능
- 디지털 서명 가능
- 속도가 느림

#### 구조 (시각화)
```
평문 → [공개키] → 암호문 → [개인키] → 평문
```

## Steganography
- 메시지를 암호화하는 것이 아니라 **메시지의 존재 자체를 숨기는 기술**입니다.

### 장점 및 단점 
- **장점**
  - 구현이 매우 쉬움
  - 겉보기에는 정상 데이터라 탐지 어려움

- **단점**
  - 많은 데이터를 숨기기 어려움 (비효율적)
  - 방법이 노출되면 재사용 불가
  - 발견되면 공격자에게 악용 가능


### 특징
- 메시지 자체를 숨김 (암호화와 차이점)
- 원문 안에 다른 의미를 숨김
- 단어나 문자 배열을 이용하여 실제 메시지 표현

### 예제
- 문자 마킹 (Character Marking)
  - 특정 글자를 연필 등으로 표시
  - 빛을 비춰야 확인 가능
 
- 보이지 않는 잉크 (Invisible Ink)
  - 열이나 화학 반응으로 나타나는 잉크 사용
 
- 핀 구멍 (Pin Punctures)
  - 종이에 작은 구멍을 뚫어 메시지 표현
  - 빛을 비춰야 확인 가능

## 치환기법 (시저암호)

- 줄리어스 시저가 사용한 암호입니다.
- 알파벳을 일정한 값(k)만큼 밀어서 암호화합니다.

### 암호화 방식

- 공식
```
C = E(p) = (p + k) mod 26

- p: 평문 문자
- k: 이동값 (Key)
- C: 암호문
```

### 예시 (Key = 3)
```
평문: meet me after the toga party

암호문: phhw ph diwhu wkh wrjd sduwb
```

## 치환기법 (다중치환)

![02](/KH_Security/암호학/01.암호의%20개념과%20고전%20암호/img/02.png)

- 2글자(쌍) 단위로 암호화하는 치환 암호입니다.
- 5x5 행렬을 이용하여 문자 변환합니다.

### 암호화 방식

- 같은 행일 시 오른쪽 문자로 치환합니다.
```
ar → RM
```


- 같은 열일 시 아래 문자로 치환합니다.
```
mu → CM
```

- 그 외에는 직사각형 형태일 때 서로 대각선 위치 문자로 교체를합니다.
```
hs → BP  
ea → IM (또는 JM)
```

## 치환기법 (다중단일문자치환)

![03](/KH_Security/암호학/01.암호의%20개념과%20고전%20암호/img/03.png)

- 하나의 규칙이 아니라 **여러 개의 치환 규칙(알파벳표)**을 사용하여 암호화합니다.
- 같은 문자라도 위치에 따라 다른 문자로 변환됩니다.

### Vigenere 방법 (표 참조)
- 예시
  - 키워드 : Infinitive
  - 평문 : I see him drive

```
평문 : I s e e h i m d r i v e
키   : I n f i n i t i v e I n
암호 : q f j m u q f l m m d r
```

### 특징
- 하나의 문자에 대해 여러 개의 암호문이 존재 → 같은 문자라도 위치마다 다르게 변환됨
- 키워드(키)에 따라 암호 방식이 계속 변함
- 문자 빈도 분석이 어려워짐 → 단일 치환보다 훨씬 분석하기 힘듦

### 단점

- 평문 구조 정보가 완전히 숨겨지지는 않음 → 완벽한 암호는 아닙니다.
- 단일 문자보다 다중 치환이라는 것은 쉽게 알 수 있습니다.
- 빈도 분석이 완전히 불가능한 것은 아님 → 반복 패턴 분석 가능합니다.
- 키워드 길이 추정 가능합니다. (핵심 취약점)

## 전치기법
- 문자를 바꾸는 게 아니라 "순서를 바꿔서" 암호화하는 방식입니다.

### 특징
- 평문의 문자 자체는 변하지 않음
- 위치만 섞어서 의미를 숨김
- 치환 암호보다 구조 기반 암호

### Rail Fence
- 문자를 지그재그로 배치 후
- 한 줄씩 읽어서 암호화
- 평문 : To have dinner in the restaurant is expensive

```
1행: t   o   h   a   v   e   d   i   n   n   e   r   i  
2행: n   t   h   e   r   e   s   t   a   u   r   a   n  
3행: t   i   s   e   x   p   e   n   s   i   v   e   a  
```
- 암호문 : tntotihhsaeevrxeepdseitnnasnuiervraeina

### 사각형 행렬 전치
- 평문을 표(행렬)에 채웁니다.
- 특정 키 순서대로 열을 읽습니다.

```
키   : 4 3 1 2 5 6 7
평문 : i h a v e t h  
       e s t u d e n  
       t s s t u d y  
       a f t e r c l  
       a s s x y z x
```

- 암호문 : `atstsvutexhssfsietaaedurytedczhnylx`

### 단점
- 빈도 분석 그대로 가능
- 구조 분석으로 해독 가능
