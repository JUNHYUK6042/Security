# 암호의 개념과 고전 암호

## 개요

- 암호는 개방된 통신망에서 데이터를 안전하게 보호하기 위해 사용하는 기술로, 네트워크 환경에서 정보 유출을 방지하기 위한 핵심 요소입니다.

## 암호를 사용하는 이유

- 네트워크는 개방되어 있어 공격에 노출되며, 평문 상태로 전송 시 정보 탈취가 가능합니다.

---

## 암호 용어

![01]()

- `평문` : 암호화 되기 전 보호되지 않은 원본 데이터입니다.  
- `암호 알고리즘(cryptographic)` : 암호를 만들어내는 시스템, 암호화키를 사용하여 평문을 암호화합니다.
- `암호화 키` : 암호문을 만들 때 암호 알고리즘에 사용되는 키입니다.
- `암호문` : 암호화 키와 암호 알고리즘을 통해 만들어낸 알 수 없는 데이터입니다.
- `암호해독` : 공격자가 암호문을 분석해서 원문을 알아내려는 시도하는 것입니다.
- `복호 알고리즘(decipher)` : 암호를 풀어 평문을 해석합니다.
- `복호화 키` : 암호화 된 데이터를 평문으로 하기 위해 복호 알고리즘에 사용되는 키입니다.

## 암호 원리

### 평문을 암호화하기 위한 연산자의 유형

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 치환 (Substitution) | 평문의 각 요소를 다른 값으로 변경 | 값 자체를 바꿈 | A → D (시저 암호) |
| 전치 (Transposition) | 평문의 요소 순서를 재배열 | 위치만 바뀜 | HELLO → LEOHL |
| 혼합 (Confusion) | 평문과 키의 관계를 복잡하게 만듦 | 키 추측 어렵게 함 | 특정 비트 선택 후 치환 |
| 확산 (Diffusion) | 평문의 변화가 암호문 전체에 퍼지도록 함 | 일부 변화 → 전체 영향 | 1비트 변경 시 전체 값 변화 |

### 사용된 키의 수

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 대칭키 (Symmetric Key) | 송신자와 수신자가 같은 키 사용 | 속도 빠름 / 키 관리 어려움 | AES, DES |
| 공개키 (Asymmetric Key) | 송신자와 수신자가 서로 다른 키 사용 | 보안성 높음 / 속도 느림 | RSA |

### 평문 처리 방식

| 구분 | 개념 설명 | 핵심 포인트 | 예시 |
|------|----------|------------|------|
| 블록 암호화 (Block Cipher) | 데이터를 일정 크기로 나눠 처리 | 정해진 크기 단위 처리 | AES (128bit 블록) |
| 스트림 암호화 (Stream Cipher) | 데이터를 연속적으로 처리 | 실시간 처리 / 빠름 | RC4 |

### 단대단 암호 (End-to-End Encryption)

- 송신자와 수신자 **종단 간에서만 암호화/복호화 수행**합니다.
- 중간 장비(라우터, 스위치)는 내용을 알 수 없습니다.
- 데이터는 전송 과정 내내 암호화 상태 유지합니다.

- 특징
  - 보안성 매우 높음
  - 중간 공격자가 내용 확인 불가능
  - 현대 인터넷 통신의 기본 방식

- 예시
  - HTTPS
  - 메신저 암호화 (카카오톡, WhatsApp 등)

### 링크 암호 (Link Encryption)

- 네트워크 **장비(라우터, 스위치 등) 단위로 암호화 수행**합니다.
- 각 구간(링크)마다 암호화/복호화 반복합니다.

- 특징
  - 구간별로 보호합니다.
  - 중간 장비에서는 평문으로 노출됩니다.
  - 장비 자체가 암호화 수행합니다.

- 예시
  - 군 통신망
  - 특정 전용망 (폐쇄망)

#### 단대단 암호나 링크 암호는 다음과 같은 이유들로 인해 가치가 없는 암호인 이유

- 링크 암호
  - 중간 장비에서 평문 노출됨
  - 내부 공격에 취약

- 단대단 암호 (과거 기준)
  - 키 관리 어려움
  - 구현 복잡
