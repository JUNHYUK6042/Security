# 비밀키 암호

---

## 비밀키 암호의 원리
- 암호화와 복호화에 "같은 키"를 사용하는 방식입니다.

![01](/KH_Security/암호학/02.비밀키%20암호/img/01.png)

### 특징
- 암호 알고리즘의 비도는 키의 비밀성에 의존합니다.
- 관용 암호, 대칭키 암호, 공유키 암호등으로 불립니다.
- 대용량 데이터 암호화에 적합합니다.

---

### 장점
- 속도가 빠릅니다.
  - 실시간 통신, 파일 암호화에 유리합니다.
- 구현이 간단합니다.
  - 시스템 부담이 적습니다.
- 대용량 데이터 처리 가능합니다.

---

### 단점
- 키를 어떻게 안전하게 전달할 것인가에 대한 분배 문제가 있습니다.
- 사용자가 많아지면 키 관리 어려워집니다.

---

### Feistel 암호 

![02](/KH_Security/암호학/02.비밀키%20암호/img/02.png)

- 블록 암호의 대표적인 구조입니다.
- 데이터를 좌(L), 우(R)로 나눠서 반복 연산합니다.
- 복호화 구조 동일합니다. (같은 구조로 키만 역순 사용)
- 라운드 함수가 완벽하지 않아도 되며, 반복할수록 강력해집니다.
- 구조적 취약점 발견이 어렵습니다.

### 대칭키 암호 종류
- DES (대표 Feistel 구조)
  - AES 등장 이후로 잘 사용하지 않습니다.
- AES
- IDEA
- SEED : 128bit 블록 암호 알고리즘
- HIGHT : 64bit 블록 암호 (IoT용)
- LEA : 128bit 블록 암호

## DES

- Feistel 구조를 사용하는 대표적인 블록 암호입니다.
- IBM의 Lucifer 알고리즘 기반으로 개발됐습니다.
- 과거 표준 암호로 사용했습니다.

### 기본 구조

- 블록 크기: 64bit씩 나누어서 처리하게 됩니다.
- 키 길이: 64bit (실제 사용 56bit - 64비트 중 8비트는 parity check로 사용하기 때문입니다.)
- 라운드 수: 16라운드
- 핵심: `64비트를 16번 나눠서 계속 섞습니다.`

#### 초기 전치

![04](/KH_Security/암호학/02.비밀키%20암호/img/04.png)

- DES는 시작하자마자 64비트 입력에 대해 초기 전치를 수행합니다.
- 초기 전치는 입력 비트의 순서를 정해진 표에 따라 다시 배열하는 과정입니다.
- 비트 값 자체를 바꾸는 것이 아니라, 위치만 바뀝니다.

---

#### f 함수 & 확장순열

![05](/KH_Security/암호학/02.비밀키%20암호/img/05.png)

- 총 64비트를 좌측 L과 우측 R로 각각 32비트씩 나눕니다
- 우측 R 32비트는 확장/순열 단계에서 48비트로 확장됩니다
- 32비트를 48비트로 확장하는 방식은 다음 그림과 같이 일부 비트를 한 번 더 사용합니다  
  즉, 일부 비트를 두 번 출력하는 방식입니다
- 확장순열일 때 표와 일부 비트는 다음과 같이 나옵니다.

![06](/KH_Security/암호학/02.비밀키%20암호/img/06.png)

---

#### 치환 선택

![07](/KH_Security/암호학/02.비밀키%20암호/img/07.png)

- 치환 서택 단계에서는 다시 32비트로 줄여야 합니다.
  그러기 위해서는 48비트를 32비트로 줄여야하는데, 이때 6비트를 4비트로 변환해야합니다.
- 모든 비트가 변환하는 과정에서 모두 참여해야합니다.

##### 6bit -> 4bit 변환하는 과정

![08](/KH_Security/암호학/02.비밀키%20암호/img/08.png)

- 101101를 예시로 변환하는 과정을 해보겠습니다.
```
- 양쪽 끝 비트인 11은 3행을 나타냅니다.
- 가운데 0110 6은 6열을 나태내며, 3행 6열의 값은 1입니다.
- 이제 1을 4bit로 변환하면 0001입니다.
```

- 그 이후로 순열과정을 거칩니다.
  - 순열과정은 `S-box를 통과한 32비트를 다시 재배열하며, 특정 비트의 영향이 전체로 퍼지게 만듭니다.`
- 순열과정 이후엔 역전치 과정을 통해 처음 했던 초기 전치(IP)를 되돌립니다.

## DES - KEY

![09](/KH_Security/암호학/02.비밀키%20암호/img/09.png)

### 초기 키 (64bit)
- DES는 64비트 키 사용용합니다.
- 실제 암호에 사용하는 키는 56비트를 사용합니다.
- 8비트는 패리티 비트(오류 검사용)입니다.
- 56bit 를 두 개로 분할합니다.
  - C0 (28bit)
  - D0 (28bit)


### PC-1 (Permuted Choice 1)
- 패리티 비트 제거한 뒤 비트 위치를 재배열 해줍니다.

### Left Shift (좌측 이동)
- 각 라운드마다 C, D를 왼쪽으로 이동합니다.
- 이동 횟수는 라운드마다 다르게 이동합니다.

### PC-2 (Permuted Choice 2)
- (C + D) = 56bit → 48bit로 축소합니다.
- 선택 + 순열 수행합니다.

## DES의 작동모드

- DES는 64bit 블록 단위로 암호화하기 때문에, 여러 블록을 처리하기 위해 다양한 모드를 사용합니다.

### ECB Mode (Electronic Codebook)

- ECB 모드는 평문을 블록 단위로 나누어 각 블록을 같은 키로 독립적으로 암호화하는 방식입니다.
- 동일한 평문 블록은 항상 동일한 암호문으로 변환됩니다.

![10](/KH_Security/암호학/02.비밀키%20암호/img/10.png)

#### 특징

- 동일한 데이터 → 동일한 암호문 생성
- 암호문에 패턴이 그대로 남음
- 구조가 단순함

#### 문제점

- 동일한 패턴이 반복되면 암호문에서도 동일하게 나타납니다.
- 공격자가 암호문 패턴을 통해 평문 구조를 유추할 수 있습니다.

---

### CBC (Cipher Block Chaining)

- CBC 모드는 이전 암호문이 다음 암호화에 영향을 주는 방식입니다.
- 현재 평문과 이전 암호문을 XOR한 후 암호화를 수행합니다.

![11](/KH_Security/암호학/02.비밀키%20암호/img/11.png)

#### 구조
```
C1 = E(P1 ⊕ IV)
C2 = E(P2 ⊕ C1)
C3 = E(P3 ⊕ C2)
```

#### 특징

- 같은 평문이라도 결과가 다르게 나옴
- 패턴이 숨겨짐

#### 장점

- ECB의 패턴 노출 문제 해결
- 보안성 증가

#### 단점

- 병렬 처리 어려움
- 하나 깨지면 다음 블록도 영향 받음

---

### CFB (Cipher Feedback)

- CFB는 이전 암호문을 입력으로 사용하여 암호화를 수행하는 방식입니다.
- 블록 암호를 스트림 암호처럼 사용할 수 있는 방식입니다.

![12](/KH_Security/암호학/02.비밀키%20암호/img/12.png)

#### 동작 원리

- 이전 암호문을 암호화
- 그 결과와 평문을 XOR하여 암호문 생성

### 특징

- 이전 암호문이 다음 암호에 영향을 줌
- 스트림 형태로 처리 가능

---

### OFB (Output Feedback)

![13](/KH_Security/암호학/02.비밀키%20암호/img/13.png) 

#### 동작 원리

- 이전 출력값을 계속 암호화
- 생성된 값과 평문을 XOR

#### 특징

- 암호문이 아닌 출력값을 피드백으로 사용
- 오류 전파가 없음
- 스트림 암호처럼 동작

---

### 3중 DES

#### 특징
- DES의 brute-force공격에 대한 취약성을 보완하는 방식입니다.
- 2개의 Key를 사용합니다.
- brute-force 공격에 DES보다 강합니다.
- 키가 다르기 때문에 중간 복호화 과정이 있어도 원문이 노출되지 않습니다.

![14](/KH_Security/암호학/02.비밀키%20암호/img/14.png)

#### 암호화
```
암호화 -> 복호화 -> 암호화
```

#### 복호화
```
복호화 -> 암호화 -> 복호화
```
