## 문제 1. 프로젝트 디렉토리를 생성하고 하부 디렉토리 및 default 파일의 소유자와 그룹 소유자를 지정하세요.

### 문제 1-1. 다음 조건에 따라 디렉토리와 계정, 그룹을 생성하세요.
  - 디렉터리 : /home/data, 그룹 : project(GID:2000), 유저 : emp(UID:2100)

```text
mkdir -p /home/data
groupadd -g 2000 project
useradd -g 2000 -u 2100 emp
```

### 문제 1-2. /home/data 디렉토리의 소유자와 그룹 소유자를 emp.project으로 지정하세요.

```text
chown emp.project /home/data
```

### 문제 1-3. /home/data 디렉토리는 소유자와 그룹 소유자는 모든 권한이 있으나 이외 사용자는 읽기만 가능하도록 설정하세요.

```text
chmod -R 775 /home/data
```

## 문제 2. Public Project 디렉토리에는 pugroup(puuser1)만 모든 권한을 가지며 이외 인원은 변경이 불가능하고 읽기만 가능하도록 설정하세요.

### 문제 2-1. 다음과 같이 public 디렉토리와 계정, 그룹을 생성하세요.
  - 디렉터리 : /home/data/public, 그룹 : pugroup(GID:3000), 유저 : puuser1(UID:3100)

```text
mkdir -p /home/data/public
groupadd -g 3000 pugroup
useradd -g 3000 -u 3100 puuser1
```

### 문제 2-2. Public 디렉토리에는 pugroup(puuser1)만 모든 권한을 가지며 이외 인원은 읽기만 가능하도록 퍼미션을 지정하세요.

```text
chown puuser1.pugroup /home/data/public
chmod -R 775 /home/data/public
```

## 문제 4. 조직내 모든 인원을 위한 down 디렉토리 생성하세요.

### 문제 4-1. 조직내 모든 사용자를 대상으로 파일 배포를 위한 디렉토리를 생성하고 퍼민션을 설정하세요.
  - 디렉터리 : /home/data/down, 그룹 : project(GID:5000), 유저 : emp(UID:5100)
  - 퍼미션 : 모든 인원이 down 접근이 가능하지만 upload는 emp만 가능

```text
mkdir -p /home/data/down
groupadd -g 5000 project
useradd -g 5000 -u 5100 emp
chown emp.projeect /home/data/down
chmod 775 /home/data/down
```

## 문제 5. 현재 emp는 홈 디렉토리 외부로 나갈 수 없다. 이때 emp가 /home/data/down 에 접근하는 방법

- emp가 down에 접근 가능하도록 설정하는 명령을 작성하세요.

```text
mkdir -p /home/emp/down
mount --bind /home/data/down /home/emp/down
```
