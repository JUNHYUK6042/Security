# Wireshark 패킷 캡쳐 & 패킷 필터

## Wireshark란
- Wireshark는 네트워크 인터페이스를 통해 흐르는 데이터를 캡처하여  
패킷 단위로 분석할 수 있게 해주는 오픈소스 네트워크 프로토콜 분석기(Network Protocol Analyzer)입니다.  
즉, 네트워크에서 실제로 어떤 데이터가 이동하는지 확인하고 분석할 수 있도록 도와주는 도구입니다.

- 주요 특징
  - 네트워크 패킷 실시간 캡처
  - 다양한 프로토콜 분석 지원
  - 패킷 내부 데이터 상세 분석
  - 필터를 통한 특정 트래픽 분석

## Wireshark 캡처 필터 종류

- **주소 필터**

| 필터 | 설명 | 예시 |
| --- | --- | --- |
| host #.#.#.# | 특정 IP 주소와 관련된 모든 패킷을 캡처한다. (출발지와 목적지 모두 포함) | host 192.168.11.12 |
| src host #.#.#.# | 특정 IP 주소에서 **출발한 패킷만** 캡처한다. | src host 192.168.11.13 |
| dst host #.#.#.# | 특정 IP 주소로 **도착하는 패킷만** 캡처한다. | dst host 192.168.11.14 |
| net #.#.#.#/# | 특정 네트워크 대역에서 발생하는 모든 패킷을 캡처한다. | net 192.168.11.0/24 |
| src net #.#.#.#/# | 특정 네트워크 대역에서 **출발한 패킷만** 캡처한다. | src net 192.168.11.0/24 |
| dst net #.#.#.#/# | 특정 네트워크 대역으로 **들어오는 패킷만** 캡처한다. | dst net 192.168.11.0/24 |
| ether host #:#:#:#:#:# | 특정 MAC 주소와 관련된 모든 패킷을 캡처한다. | ether host aa:bb:cc:dd:ee:ff |
| ether src #:#:#:#:#:# | 특정 MAC 주소에서 **출발한 패킷만** 캡처한다. | ether src aa:bb:cc:dd:ee:ff |
| ether dst #:#:#:#:#:# | 특정 MAC 주소로 **도착하는 패킷만** 캡처한다. | ether dst aa:bb:cc:dd:ee:ff |

- **포트 필터**

| 필터 | 설명 | 예시 |
|---|---|---|
| port ## | 특정 포트 번호와 관련된 모든 패킷을 캡처한다. (출발지와 목적지 포트 모두 포함) | port 80 |
| src port ## | 특정 포트에서 **출발한 패킷만** 캡처한다. | src port 80 |
| dst port ## | 특정 포트로 **들어오는 패킷만** 캡처한다. | dst port 21 |
| tcp port ## | 특정 TCP 포트를 사용하는 패킷만 캡처한다. | tcp port 53 |
| udp port ## | 특정 UDP 포트를 사용하는 패킷만 캡처한다. | udp port 53 |
| portrange ##-## | 특정 범위의 포트를 사용하는 패킷을 캡처한다. | portrange 20-25 |

- **프로토콜**
```
tcp, udp, icmp, icmp6, ipx, ip, ip6, arp, rarp  ....
```

- **다중 필터**
```
and, or
```

## Wireshark 디스플레이 필터

| 연산자 | 설명 | 예시 |
|---|---|---|
| == | 특정 값과 같은 패킷만 필터링 | ip.src == 192.168.0.10 |
| === | 엄격하게 동일한 값을 비교 (데이터 타입까지 동일) | tcp.port === 80 |
| != | 특정 값과 다른 패킷 필터링 | ip.src != 192.168.0.10 |
| !== | 엄격하게 다른 값 비교 | tcp.port !== 80 |
| < | 특정 값보다 작은 값 필터링 | frame.len < 100 |
| > | 특정 값보다 큰 값 필터링 | frame.len > 1000 |
| <= | 특정 값보다 작거나 같은 값 필터링 | tcp.port <= 1024 |
| >= | 특정 값보다 크거나 같은 값 필터링 | tcp.port >= 1024 |
| is present | 해당 필드가 패킷에 존재하는지 확인 | tcp.options.mss is present |
| contains | 특정 문자열이나 데이터가 포함된 패킷 필터링 | http.host contains "google" |
| in | 특정 값 목록(list)에 포함되는 패킷 필터링 | tcp.port in {80 443 8080} |

## Wireshark 패킷 분석

![07]()

- 해당 패킷을 확인해 보면 TCP Flags 항목에 **SYN과 ACK 플래그**가 설정되어 있습니다.
- 이는 TCP 연결 과정에서 발생하는 **3-Way Handshake의 두 번째 단계(SYN-ACK)** 패킷입니다.

- TCP 3-Way Handshake 과정은 다음과 같습니다.
```
1단계  
Client → Server  
SYN

2단계  
Server → Client  
SYN + ACK

3단계  
Client → Server  
ACK
```
- 위 과정이 완료되면 TCP 연결이 성립되고 이후 HTTP 통신이 시작됩니다.
