# Wireshark 패킷의 이해 (Transport Layer)

## 개요

- Transport Layer는 Host 간 데이터 전달을 담당하는 계층이다.  
- Network Layer(IP)가 패킷을 목적지까지 전달하는 역할을 한다면,  
Transport Layer는 데이터 전송의 신뢰성, 흐름 제어, 오류 검출 등의 기능을 수행합니다.

- 주요 기능
  - 데이터 흐름 제어 (Flow Control)
  - 오류 검출 (Checksum)
  - 연결 설정 및 해제
  - 데이터 재전송
  - 데이터 순서 보장


- 대표 프로토콜
  - TCP
  - UDP

---

## TCP (Transmission Control Protocol)

- TCP는 신뢰성 있는 데이터 전송을 제공하는 연결지향 프로토콜이다.

- 특징
  - 연결지향 방식
  - 신뢰성 있는 데이터 전달
  - 데이터 순서 보장
  - 흐름 제어
  - 재전송 기능 제공
 
### TCP Connection Initialization (Three-Way Handshake)

- TCP는 데이터를 전송하기 전에 송신자(sender)와 수신자(receiver) 사이에 connection을 먼저 설정합니다.  
연결이 생성된 이후에 data segment 전송이 시작됩니다.

- TCP 연결이 초기화될 때 다음 정보들이 설정됩니다.
  - Sequence Number (Seq#)
    - 데이터의 순서를 관리하기 위한 번호입니다.
    - 각 호스트는 자신의 초기 sequence number를 설정합니다.
  - Buffer : `네트워크에서 받은 데이터를 임시로 저장하는 메모리 공간입니다.`
    - 수신된 데이터를 임시로 저장하는 공간입니다.
    - 데이터가 순서대로 도착하지 않을 경우 버퍼에 저장됩니다.
  - Flow Control 정보 : `Receiver의 버퍼 상태를 기준으로 Sender의 전송 속도를 조절하여 데이터 손실을 방지하는 메커니즘입니다.`
    - 수신된 데이터를 임시로 저장하는 공간입니다.
    - 데이터가 순서대로 도착하지 않을 경우 버퍼에 저장됩니다.

#### Three-Way Handshake

- TCP 연결은 3단계 과정으로 생성됩니다.

##### Step 1
```
Client → Server (SYN segment 전송)

- client가 server에게 연결 요청합니다.
- client의 초기 sequence number 설정합니다.
- 데이터는 포함되지 않습니다.
```

##### Step 2
```
Server → Client (SYN + ACK 전송)

- server가 SYN 요청을 수신
- client 요청 승인
- server의 초기 sequence number 설정
- buffer 및 연결 변수 할당
```

##### Step 3
```
Client → Server (ACK 전송)

- client가 SYNACK를 수신하고 ACK 응답
- client의 buffer 및 연결 변수 할당
- 이후 데이터 전송 가능
```

## TCP Header 구조

TCP 기본 헤더 크기

20 byte

필드 구조

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| Source Port | 2 byte | 송신 애플리케이션 포트 |
| Destination Port | 2 byte | 목적지 애플리케이션 포트 |
| Sequence Number | 4 byte | 전송 데이터 시작 번호 |
| Acknowledgment Number | 4 byte | 다음에 받을 데이터 번호 |
| Header Length | 4 bit | TCP 헤더 길이 |
| Reserved | 6 bit | 예약 필드 |
| Flags | 6 bit | TCP 제어 플래그 |
| Window Size | 2 byte | 수신 가능한 데이터 크기 |
| Checksum | 2 byte | 오류 검출 |
| Urgent Pointer | 2 byte | 긴급 데이터 위치 |
| Options | variable | 추가 옵션 |

---

## TCP Flags

TCP는 연결 제어를 위해 여러 플래그를 사용한다.

| Flag | 설명 |
|-----|-----|
| SYN | 연결 요청 |
| ACK | 응답 확인 |
| FIN | 연결 종료 |
| RST | 연결 재설정 |
| PSH | 데이터를 즉시 전달 |
| URG | 긴급 데이터 표시 |

---

## UDP (User Datagram Protocol)

UDP는 비연결형 전송 프로토콜이다.

특징

- 연결 설정 과정 없음
- 빠른 데이터 전송
- 신뢰성 보장 없음
- 재전송 기능 없음
- 패킷 손실 가능

---

## UDP Header 구조

UDP 헤더 크기

8 byte

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| Source Port | 2 byte | 송신 포트 |
| Destination Port | 2 byte | 목적지 포트 |
| Length | 2 byte | UDP 데이터그램 전체 길이 |
| Checksum | 2 byte | 오류 검출 |

---

## TCP와 UDP 차이

TCP

- 연결지향
- 신뢰성 보장
- 재전송 존재
- 흐름 제어 존재
- 속도 상대적으로 느림

UDP

- 비연결
- 신뢰성 없음
- 재전송 없음
- 흐름 제어 없음
- 속도 빠름

---

## 정리

Transport Layer는 Host 간 데이터 전달의 신뢰성과 흐름을 관리하는 계층이며, TCP는 신뢰성 중심의 프로토콜이고 UDP는 속도 중심의 프로토콜이다.
