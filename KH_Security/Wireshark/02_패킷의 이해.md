# Wireshark 패킷의 이해

## 개요
- Wireshark는 네트워크를 통해 전달되는 패킷을 캡쳐하고 분석하는 도구입니다.
- 패킷 분석을 위해서는 Ethernet Frame, IP Header, TCP/UDP Header 등의 구조를 이해해야 합니다.

---

## 대표적인 프로토콜 헤더 크기

- TCP Header : 20 byte  
- UDP Header : 8 byte  
- IP Header : 20 byte

- 패킷은 다음과 같은 계층 구조로 이루어진다.
```text
Application Layer  
Transport Layer (TCP / UDP)  
Network Layer (IP / ARP / ICMP)  
Link Layer (Ethernet)  
```

---

## Ethernet Frame 구조

- Ethernet Frame은 실제 네트워크에서 데이터가 전달되는 기본 프레임 구조이다.

### 구조

| 구분 | 설명 | 예시 |
|---|---|---|
| Preamble (8 byte) | 프레임의 시작을 알리고 송수신 장비 간 동기화를 맞추는 필드 (Wireshark에서는 캡쳐되지 않음) | `10101010` 패턴 |
| Destination MAC Address (6 byte) | 목적지 장비의 MAC 주소 | `00:1A:2B:3C:4D:5E` |
| Source MAC Address (6 byte) | 패킷을 전송한 장비의 MAC 주소 | `00:0C:29:AB:CD:EF` |
| Type (2 byte) | 상위 계층 프로토콜 종류를 나타내는 필드 | `0x0800 = IPv4`, `0x0806 = ARP`, `0x86DD = IPv6` |
| Data (Datagram) | 상위 계층 데이터(IP Datagram)가 들어가는 영역 | `IP Header + TCP Header + Data` |
| FCS (4 byte) | Frame Check Sequence, 프레임 오류 검출을 위한 값 (대부분 Wireshark에서 캡쳐되지 않음) | `CRC32 값` |

## IP Header 구조

IP는 네트워크 계층에서 사용되는 프로토콜로  
패킷을 목적지까지 전달하는 역할을 한다.

IP Header 구조

Version (4 bit)  
IP 프로토콜 버전  
IPv4 = 4

Header Length (4 bit)  
IP Header의 길이

Type Of Service (8 bit)  
서비스 우선순위 및 QoS 정보

Total Length (16 bit)  
IP Datagram 전체 길이  
(IP Header + Data)

Identification (16 bit)  
Fragment된 패킷을 재조립하기 위한 식별 값

Flags (3 bit)  
Fragment 여부를 나타내는 필드

Fragment Offset (13 bit)  
Fragment된 데이터의 위치

TTL (8 bit)  
Time To Live  
라우터를 지날 때마다 1씩 감소

Protocol (8 bit)  
상위 프로토콜 종류

1  : ICMP  
6  : TCP  
17 : UDP  

Header Checksum (16 bit)  
IP Header 오류 검출

Source IP Address (32 bit)  
출발지 IP 주소

Destination IP Address (32 bit)  
목적지 IP 주소


# Identification

Fragment된 패킷을 같은 Datagram으로 식별하기 위한 값이다.

IP는 신뢰적인 데이터 전송을 하지 않지만  
패킷이 Fragment 되었을 때 재조립하기 위해 Identification 값을 사용한다.

패킷이 Router를 지나가는 과정에서  
Datagram이 너무 크면 Router가 패킷을 여러 개로 나눌 수 있다.

예

원래 Datagram

DATA

Fragment 발생

DATA1  
DATA2

이때 두 패킷은 동일한 Identification 값을 가진다.

Destination Host는 이 값을 이용하여  
Fragment된 패킷들을 다시 하나의 Datagram으로 재조립한다.


# TCP Sequence Number와 차이

TCP Sequence Number

신뢰적인 데이터 전송을 위해 사용한다.

IP Identification

Fragment된 Datagram을 재조립하기 위해 사용한다.


# Flags (3 bit)

Fragment 여부를 나타낸다.

1bit  
사용하지 않음

2bit (DF : Don't Fragment)

1 : Fragment 하지 말 것  
0 : Fragment 가능

3bit (MF : More Fragment)

1 : 뒤에 같은 Identification을 가진 Fragment가 더 있음  
0 : 마지막 Fragment

Fragment가 없는 경우

Flags = 000


# Fragment Offset

Fragment된 Datagram이  
원래 데이터에서 몇 번째 위치인지 나타내는 값이다.

예

원본 데이터

DATA DATA DATA DATA

Fragment 발생

Fragment1  
Fragment2  
Fragment3

이때

Identification 값은 동일하고  
Fragment Offset 값은 서로 다르다.

Destination Host는 Fragment Offset을 이용하여  
패킷을 원래 순서대로 재조립한다.

※ Fragment 재조립은 Router가 아니라 Destination Host에서 수행한다.


# TTL (Time To Live)

패킷이 네트워크에서 살아 있을 수 있는 시간을 의미한다.

패킷이 Router를 하나 지날 때마다

TTL = TTL - 1

TTL 값이 0이 되면 패킷은 폐기된다.

목적

라우팅 과정에서 무한 루프 방지


# IP Address

Source IP  
패킷을 보낸 호스트의 IP 주소

Destination IP  
패킷을 받을 호스트의 IP 주소


# ARP (Address Resolution Protocol)

ARP는 IP 주소를 MAC 주소로 변환하는 프로토콜이다.

예

192.168.0.10

이라는 IP 주소를 가진 장비의

MAC 주소를 알아내기 위해 사용된다.


# ARP 동작 과정

1. 송신 Host가 목적지 IP로 통신하려고 한다.
2. ARP Broadcast를 네트워크에 전송한다.
3. 해당 IP를 가진 Host가 ARP Reply를 전송한다.
4. 송신 Host는 MAC 주소를 ARP Cache에 저장한다.
5. 이후 통신은 해당 MAC 주소를 이용하여 수행된다.


# ARP Header 구조

Hardware Type (2 byte)

네트워크 타입

Ethernet = 1


Protocol Type (2 byte)

사용 프로토콜

IPv4 = 0x0800


Hardware Length (1 byte)

MAC 주소 길이

Ethernet = 6 byte


Protocol Length (1 byte)

IP 주소 길이

IPv4 = 4 byte


OP (Operation)

ARP 요청 / 응답 구분

1 : ARP Request  
2 : ARP Reply  


Sender Address

Sender MAC Address  
Sender IP Address


Target Address

Target MAC Address  
Target IP Address
