# 04_04_침입 방지 시시템 실습 2

## 개요

- 이번 실습은 Snort Rule을 이용하여 threshold 옵션, 사설 IP 탐지,   
  TCP Scan 탐지, ICMP 탐지, Land Attack, Flooding 공격 탐지를 확인하는 과정입니다.
- 단순히 패킷 내용을 보는 것이 아니라 반복 횟수,  
  플래그 값, 출발지/목적지 IP, 공격 패턴을 기준으로 침입을 탐지합니다.
- IDS 서버에서 Snort를 실행한 뒤 공격 트래픽을 발생시키고 alert 로그를 통해 탐지 여부를 확인합니다.

---

## 공통 실행 명령어

- 모든 실습은 먼저 IDS 서버에서 아래 명령어를 실행한 후 진행합니다.
```
snort -i ens160 -A fast -d -c /etc/snort/snort.conf
```

- 모든 Rule 설정도 다음과 같은 명령어로 파일에 적혀있는 Rule을 수정합니다.
```
vi /etc/snort/rules/local.rules
```

- 이후 공격 또는 테스트 트래픽을 발생시키고 아래 명령어로 탐지 결과를 확인합니다.
```
cat /var/log/snort/alert
```

## Threshold 실습

### Rule 설정
```
alert icmp any any -> any any (msg:"[THRESHOLD] ICMP Echo Request"; itype:8; threshold:type threshold, track by_dst, count 3, seconds 10; sid:2001001; rev:1;)
alert icmp any any -> any any (msg:"[LIMIT] ICMP Echo Request"; itype:8; threshold:type limit, track by_dst, count 3, seconds 10; sid:2001002; rev:1;)
alert icmp any any -> any any (msg:"[BOTH] ICMP Echo Request"; itype:8; threshold:type both, track by_dst, count 3, seconds 10; sid:2001003; rev:1;)
```

![63](/KH_Security/Linux/Firewall/img/63.png)

#### 의미

- threshold  
  → count 조건을 만족할 때마다 주기적으로 alert 발생
- limit  
  → 일정 시간 동안 최대 count 횟수까지만 경고 발생
- both  
  → threshold + limit를 함께 적용
- by_dst  
  → 목적지 IP 기준으로 탐

- 세 Rule 모두 ICMP Echo Request (ping 요청, itype:8)를 탐지합니다.
- 목적지 기준(track by_dst)으로 동일 대상에게 반복되는 ping 요청을 감시합니다.
- count 3, seconds 10 설정은 10초 동안 3번의 이벤트를 기준으로 동작합니다.

### 결과 확인

- 다음 명령어는 192.168.12.11로 ICMP Echo Request를 10번 전송합니다.
```
ping -c 10 192.168.12.11
```

![64](/KH_Security/Linux/Firewall/img/64.png)

- 앞에서 설정한 Rule은 ICMP Echo Request(itype:8)를 대상으로 하고, `count 3, seconds 10` 조건을 사용합니다.
- 따라서 10초 안에 ping 요청을 여러 번 보내면  
  threshold, limit, both 조건이 각각 어떻게 alert를 발생시키는지 확인할 수 있습니다.
- 여기서 중요한 부분은 ping 성공 여부가 아니라  
  Snort가 반복된 ICMP 요청을 threshold 조건에 따라 탐지했는지 확인하는 것입니다.

## 사설 IP 대역 탐지

### Rule 설정

- 다음 Rule은 사설 IP 대역인 192.168.0.0/16에서 발생하는 ICMP 트래픽을 탐지하는 Rule입니다.
```
alert icmp 10.0.0.0/8 any -> any any (msg:"Bogon ip (A)"; sid:1000001;)
alert icmp 172.16.0.0/16 any -> any any (msg:"Bogon ip (B)"; sid:1000002;)
alert icmp 192.168.0.0/16 any -> any any (msg:"Bogon ip (C)"; sid:1000003;)
```

### 결과 확인

- EXTERNAL 서버(192.168.11.19)에서 AP 서버(192.168.12.11)로 ping을 보냅니다.
```
ping 192.168.12.11
```

![66](/KH_Security/Linux/Firewall/img/66.png)

- 다음과 같이 로그를 확인합니다.

```
cat /var/log/snort/alert
```
![67](/KH_Security/Linux/Firewall/img/67.png)

- alert 로그에서 `Bogon ip (C)`가 정상적으로 탐지된 것을 확인할 수 있습니다.
- 출발지 192.168.11.19는 사설 IP 대역이고 목적지 192.168.12.11 역시 내부망 주소입니다.
- 따라서 Snort는 해당 ICMP 요청을 사설 IP 기반 접근으로 판단하여 탐지한 상태입니다.

## flag 값을 통한 TCP 스캔 탐색

### FIN Scan 탐지

#### Rule 설정

- 다음 Rule은 FIN 플래그만 설정된 TCP 패킷을 탐지하는 Rule입니다.
```
alert tcp any any -> any any (msg:"FIN Scan Detected"; flags:F; flow:stateless; sid:1000102; rev:1;)
```

```

```

- 다음과 같이 로그를 확인합니다.

![68](/KH_Security/Linux/Firewall/img/68.png)

- 출발지 192.168.11.36에서 목적지 192.168.12.11의 여러 포트로 FIN 플래그만 가진 TCP 패킷이 반복적으로 전달되었습니다.
- 이는 정상적인 서비스 접속이 아니라 열려 있는 포트를 확인하기 위한 스캔 동작입니다.

### XMAS Scan 탐지

#### Rule 설정

- 다음 Rule은 XMAS Scan 패킷을 탐지하는 Rule입니다.
```
alert tcp any any -> any any (msg:"XMAS Scan Detected"; flags:FPU; flow:stateless; sid:1000101; rev:1;)
```

- `flags:FPU`는 FIN(F), PSH(P), URG(U) 플래그가 동시에 설정된 패킷을 의미합니다.
- 이러한 패킷은 정상적인 통신보다는 포트 스캔 목적으로 자주 사용됩니다.

#### 결과 확인

- Kali에서 다음 명령어로 XMAS Scan을 수행합니다.
```
nmap -sX 192.168.12.11
```

- 다음과 같이 로그를 확인합니다.

![70](/KH_Security/Linux/Firewall/img/70.png)

- alert 로그에서 `XMAS Scan Detected`가 정상적으로 탐지된 것을 확인할 수 있습니다.
- 출발지 192.168.11.36에서 목적지 192.168.12.11의 여러 포트로  
  비정상적인 TCP 플래그를 가진 패킷이 전달되었습니다.

## Land Attack 탐지

### Rule 설정

- 다음 Rule은 출발지 IP와 목적지 IP가 동일한 패킷을 탐지하는 Rule입니다.
```
alert icmp any any -> 192.168.11.0/24 any (msg:"Land Attack Detect"; sameip; sid:20000001;)
```

- `sameip` 옵션은 source IP와 destination IP가 같은 경우를 의미합니다.
- 정상적인 통신에서는 거의 발생하지 않으며 대표적인 DoS 공격 유형인 Land Attack에서 사용됩니다.

### 결과 확인

- Kali에서 다음 명령어로 Land Attack을 수행합니다.
```
hping3 --icmp -a 192.168.11.19 192.168.11.19
```

- 다음과 같이 로그를 확인합니다.

![72](/KH_Security/Linux/Firewall/img/72.png)

- alert 로그에서 `Land Attack Detect`가 정상적으로 탐지된 것을 확인할 수 있습니다.
- 출발지와 목적지가 모두 192.168.11.19로 동일합니다.
- 이는 정상적인 통신이 아니라 자기 자신에게 패킷을 보내는 비정상적인 공격 형태입니다.

## Flooding 탐지

### SYN Flooding 탐지

#### Rule 설정

- 다음 Rule은 짧은 시간 안에 대량의 SYN 패킷이 발생하는 SYN Flooding 공격을 탐지하는 Rule입니다.
```
alert tcp any any -> any 80 (msg:"ATT : SYN Flooding"; flags:S; threshold:type threshold, track by_dst, count 100, seconds 1; sid:2000002;)
```

- `flags:S`는 SYN 플래그를 의미합니다.
- threshold 옵션을 사용하여 1초 동안 동일 목적지로 100개 이상의 SYN 요청이 발생하면 탐지합니다.

### 결과 확인

- Kali에서 다음 명령어로 SYN Flooding 공격을 수행합니다.
```
hping3 -S -p 80 --flood 192.168.12.11
```

- `-S`는 SYN 플래그 설정
- `-p 80`은 HTTP 서비스 포트 대상
- `--flood`는 가능한 빠르게 대량의 패킷 전송을 의미합니다.

- 다음과 같이 로그를 확인합니다.

![73](/KH_Security/Linux/Firewall/img/73.png)

- alert 로그에서 `ATT : SYN Flooding`이 정상적으로 탐지된 것을 확인할 수 있습니다.
- 출발지 192.168.11.36에서 목적지 192.168.12.11의 80번 포트로 대량의 SYN 요청이 반복적으로 전달되었습니다.
- 동시에 `SYN Scan Detected`도 함께 발생할 수 있습니다.
- 이는 Flooding 과정 자체가 다량의 SYN 요청이기 때문입니다.

## UDP Flooding 탐지

### Rule 설정

- 다음 Rule은 짧은 시간 안에 대량의 UDP 패킷이 발생하는 UDP Flooding 공격을 탐지하는 Rule입니다.
```
alert udp any any -> any any (msg:"ATT : UDP Flooding"; threshold:type threshold, track by_dst, count 100, seconds 1; sid:2000003;)
```

- threshold 옵션을 사용하여 1초 동안 동일 목적지로 100개 이상의 UDP 패킷이 발생하면 탐지합니다.
- UDP는 연결 과정 없이 바로 패킷을 전송하므로 대량 전송 시 서비스 마비 공격에 자주 사용됩니다.

### 결과 확인

- Kali에서 다음 명령어로 UDP Flooding 공격을 수행합니다
```
hping3 --udp --flood -p 53 192.168.12.11
```

- `--udp`는 UDP 패킷 전송
- `-p 53`은 DNS 서비스 포트 대상
- `--flood`는 가능한 빠르게 대량의 패킷 전송을 의미합니다.

- 다음과 같이 로그를 확인합니다.

![74](/KH_Security/Linux/Firewall/img/74.png)

- alert 로그에서 `ATT : UDP Flooding`이 정상적으로 탐지된 것을 확인할 수 있습니다.
- 출발지 192.168.11.36에서 목적지 192.168.12.11의 53번 포트(DNS 서비스)로  
  대량의 UDP 패킷이 반복적으로 전달되었습니다.
- 이는 정상적인 DNS 요청이 아니라 서비스 자원을 고갈시키기 위한 공격 형태입니다.

## ICMP Flooding 탐지

### Rule 설정

- 다음 Rule은 짧은 시간 안에 대량의 ICMP 패킷이 발생하는 ICMP Flooding 공격을 탐지하는 Rule입니다.
```
alert icmp any any -> any any (msg:"ATT : ICMP Flooding"; threshold:type threshold, track by_dst, count 100, seconds 1; sid:2000004;)
```

- threshold 옵션을 사용하여 1초 동안 동일 목적지로 100개 이상의 ICMP 요청이 발생하면 탐지합니다.
- ICMP Flooding은 대량의 ping 요청을 보내 서버 자원을 소모시키는 대표적인 DoS 공격입니다.

### 결과 확인

- Kali에서 다음 명령어로 ICMP Flooding 공격을 수행합니다.
```
hping3 --icmp --flood 192.168.12.11
```

- `--icmp`는 ICMP 패킷 전송
- `--flood`는 가능한 빠르게 대량의 패킷 전송을 의미합니다.

- 다음과 같이 로그를 확인합니다.

![75](/KH_Security/Linux/Firewall/img/75.png)

- alert 로그에서 `ATT : ICMP Flooding`이 정상적으로 탐지된 것을 확인할 수 있습니다.
- 동시에 `ICMP Ping` 탐지도 함께 발생합니다.
- Flooding 공격 자체가 대량의 ICMP 요청이기 때문에 일반 ICMP 탐지 Rule에도 함께 매칭됩니다.

## 요약 정리

- threshold는 반복 이벤트를 제어하는 핵심 옵션입니다.
- Bogon IP는 비정상적인 사설 IP 접근을 탐지합니다.
- XMAS / FIN / SYN Scan은 대표적인 포트 스캔 공격입니다.
- ICMP 탐지는 일반 ping과 특수 ICMP 요청을 확인합니다.
- Land Attack은 출발지와 목적지가 동일한 비정상 공격입니다.
- SYN / UDP / ICMP Flooding은 대량 트래픽 기반 DoS 공격입니다.
