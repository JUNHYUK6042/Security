# Linux Firewall (Zone)

## 개요

- 리눅스 방화벽은 ipchains에서 netfilter 구조로 발전했습니다.
- iptables는 netfilter의 사용자 인터페이스였지만 명령어가 복잡했습니다.
- CentOS7부터 firewalld와 firewall-cmd가 도입되어 설정이 간편해졌습니다.
- firewalld는 서비스(daemon) 기반으로 동작하며 zone과 service 중심으로 정책을 구성합니다.

---

## firewalld 기본 개념

### 방화벽 동작 방식

- firewalld는 systemctl로 관리되는 서비스입니다.
- 설정은 즉시 적용 방식과 영구 적용 방식으로 나뉩니다.

### 주요 명령어

| 명령어 | 설명 |
|--------|------|
| systemctl start firewalld | 방화벽 시작입니다 |
| systemctl stop firewalld | 방화벽 종료입니다 |
| systemctl enable firewalld | 부팅 시 자동 실행 설정입니다 |
| systemctl is-active firewalld | 실행 상태 확인입니다 |
| firewall-cmd --reload | 설정 재적용입니다 |

### 주요 설정 파일 구조

| 경로 | 설명 |
|------|------|
| /usr/lib/firewalld | 기본 설정 저장 위치 |
| /etc/firewalld | 사용자 설정 저장 위치 |

### 특징

- `/usr/lib/firewalld`는 기본 정의 파일로 수정하면 안 됩니다.
- `/etc/firewalld`는 변경된 설정이 저장되는 위치입니다.
- firewall-cmd로 설정한 내용은 `/etc/firewalld`에 반영됩니다.

### 설정 파일 확인

#### 기본 설정 파일

- 다음 명령어로 firewalld의 기본 설정 파일 위치를 확인합니다.
```
  ls /usr/lib/firewalld
```
![01](/KH_Security/Linux/Firewall/img/01.png)

- 해당 경로는 firewalld의 기본 정책이 저장된 위치입니다.
- 기본적으로 제공되는 zone, service, icmp, policy 등이 포함되어 있습니다.
- 이 경로의 파일은 시스템 기본값이므로 직접 수정하면 안 됩니다.

---

#### 사용자 설정 파일

- 다음 명령어로 사용자 설정 파일 위치를 확인합니다.
```
  ls /etc/firewalld
```
![02](/KH_Security/Linux/Firewall/img/02.png)

- 해당 경로는 사용자가 변경하거나 추가한 설정이 저장되는 위치입니다.
- firewall-cmd 명령어로 설정한 내용이 이 경로에 반영됩니다.
- 기본 설정을 덮어쓰는 구조로 동작합니다.

## Zone

- Zone은 서버 역할에 맞는 보안 정책 묶음이며, 서버의 용도에 맞게 미리 정의된 네트워크 보안 레벨을 의미합니다.
- 서비스 기반으로 접근 제어를 구성합니다.

### 기본 Zone 종류
```text
  block, dmz, drop, external, home, internal, public, trusted, work
```

---

### public zone 설정 확인

- 다음 명령어로 public zone 설정을 확인할 수 있습니다.
```
  cat /usr/lib/firewalld/zones/public.xml
```

![03](/KH_Security/Linux/Firewall/img/03.png)

#### 주요 항목 설명

| 항목 | 설명 |
|------|------|
| `<zone>` | 하나의 방화벽 정책 단위를 의미합니다 |
| `<short>` | zone의 이름입니다 |
| `<description>` | 해당 zone의 보안 수준 및 용도를 설명합니다 |
| `<service name="ssh"/>` | SSH(22번 포트) 접근을 허용합니다 |
| `<service name="dhcpv6-client"/>` | DHCPv6 클라이언트 통신을 허용합니다 |
| `<service name="cockpit"/>` | 웹 기반 관리 도구(cockpit) 접근을 허용합니다 |

---

### Default Zone

- 반드시 하나만 존재합니다.
- `/etc/firewalld/firewalld.conf`에서 설정됩니다.
- 인터페이스가 지정되지 않으면 default zone이 적용됩니다.

![04](/KH_Security/Linux/Firewall/img/04.png)

---

### Zone 활성화

- Interface별 로 한 개만 활성화 가능합니다.
- Default zone의 경우 인터페이스 할당과 무관하게 활성화됩니다.
- zone이 할당되지 않은 interface에는 default zone이 적용됩니다.
- Interface에 지정되지 않은 zone에 --add-source가 설정된 경우 interfac에 활성화 됩니다.
- Interface에 지정되지 않은 zone에 service나 port만 설정한 경우 저장은 되지만 활성화 되지 않습니다.

---

### Firewall-cmd 주요 옵션

| 옵션 | 예시 | 설명 |
|------|------|------|
| --get-zones | firewall-cmd --get-zones | 전체 zone 목록을 출력 |
| --list-all-zones | firewall-cmd --list-all-zones | 모든 zone의 설정 내용을 출력 |
| --list-all | firewall-cmd --list-all | default zone의 설정 내용을 출력 |
| --list-all --zone=<zone> | firewall-cmd --list-all --zone=public | 특정 zone의 설정 내용을 출력 |
| --info-zone=<zone> | firewall-cmd --info-zone=public | 지정한 zone의 상세 정보를 출력 |
| --get-default-zone | firewall-cmd --get-default-zone | 현재 default zone을 확인 |
| --set-default-zone=<zone> | firewall-cmd --set-default-zone=dmz | default zone을 변경 |
| --get-active-zone | firewall-cmd --get-active-zone | 인터페이스별로 활성화된 zone을 확인 |
| --permanent --new-zone=<zone> | firewall-cmd --permanent --new-zone=testzone | 새로운 zone을 생성 |
| --permanent --delete-zone=<zone> | firewall-cmd --permanent --delete-zone=testzone | zone을 삭제 |

#### zone 상세 설정 확인
```
  firewall-cmd --list-all
```

![08](/KH_Security/Linux/Firewall/img/08.png)

##### 주요 항목 해석

| 항목 | 설명 |
|------|------|
| public (active) | 현재 활성화된 zone입니다 |
| target: default | 기본 정책을 따릅니다 |
| icmp-block-inversion: no | ICMP 차단 반전 기능이 비활성화 상태입니다 |
| interfaces: ens160 | 해당 zone이 적용된 네트워크 인터페이스입니다 |
| sources: | 특정 IP 지정 없음 상태입니다 |
| services: cockpit dhcpv6-client ssh | 허용된 서비스 목록입니다 |

### 인터페이스 Zone 할당

- 네트워크 인터페이스에 zone을 할당하여 방화벽 정책을 적용합니다.
- default zone 대신 특정 zone을 인터페이스에 직접 지정할 수 있습니다.
 

#### 명령어 형식
```
  firewall-cmd --zone=<zone> --change-interface=<interface>
```

#### 옵션 정리

| 옵션 | 예시 | 설명 |
|------|------|------|
| --zone | --zone=dmz | 적용할 zone을 지정합니다 |
| --change-interface | --change-interface=ens160 | 해당 인터페이스에 zone을 적용합니다 |

---

#### 적용 예시

- 다음 명령어로 ens160 인터페이스에 dmz zone을 적용합니다.
```
  firewall-cmd --zone=dmz --change-interface=ens160
```

## 정리

- firewalld는 zone 기반으로 동작하며, 인터페이스에 zone을 할당하여 방화벽 정책을 적용합니다.
- `/usr/lib/firewalld`는 기본 설정, `/etc/firewalld`는 사용자 설정이며 사용자 설정이 우선 적용됩니다.
- default zone은 기본값이며, 실제 적용되는 것은 인터페이스에 연결된 active zone입니다.
- `firewall-cmd --list-all`로 현재 허용된 서비스(열린 포트)를 확인할 수 있습니다.
- 필요한 서비스만 허용하는 방식으로 동작하며, zone 변경 시 보안 정책이 즉시 달라집니다.
