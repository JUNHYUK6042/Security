# Snort IDS 설치 및 탐지 실습

## 개요

- IDS(Intrusion Detection System)는 침입을 탐지하는 시스템입니다.
- Snort는 대표적인 오픈소스 IDS이며 A-BOX 역할을 담당합니다.
- 패킷을 분석하여 공격이나 비정상 접근을 탐지하고 로그를 남깁니다.

---

## 구조

- A-BOX : Snort (탐지 엔진)
- E-BOX : DAQ (패킷 수집)
- D-BOX : Barnyard2, MySQL (로그 저장)
- R-BOX : IPS (차단 및 대응 강화)

### A-BOX (Analyzer Box)

- Snort가 동작하는 핵심 탐지 영역입니다.
- E-BOX에서 전달받은 패킷을 분석합니다.
- Rule과 비교하여 공격 여부를 판단합니다.
- 침입, 비정상 트래픽, 정책 위반 등을 탐지합니다.

---

### E-BOX (Event Box)

- DAQ(Data Acquisition)가 동작하는 영역입니다.
- 네트워크에서 실제 패킷을 수집합니다.
- 수집한 패킷을 A-BOX(Snort)로 전달합니다.

---

### D-BOX (Database Box)

- Barnyard2, MySQL 등이 사용하는 영역입니다.
- Snort가 탐지한 침입 이벤트와 로그를 저장합니다.
- 공격 발생 시간, 출발지 IP, 목적지 IP, 사용된 프로토콜 등을 기록합니다.
- 저장된 로그를 통해 공격 상황을 다시 확인하고 분석할 수 있습니다.
- 침입 과정을 재현하거나 사고 분석(포렌식)을 위해 중요합니다.

---

### R-BOX (Response Box)

- IPS(Intrusion Prevention System) 역할을 합니다.
- 탐지된 공격에 대해 실제 차단이나 대응을 수행합니다.
- 방화벽 연동, 패킷 차단, 세션 종료 등이 가능합니다.
- A-BOX보다 더 적극적인 보안 대응을 수행합니다.

---

## Snort 설치 과정

### 관련 패키지 설치

- Snort는 설치 전에 필요한 라이브러리와 개발 패키지를 먼저 설치합니다.
- Snort는 컴파일과 패킷 분석에 필요한 여러 라이브러리를 사용합니다.
- Rocky Linux에서는 powertools 저장소 활성화가 필요합니다.
```
- dnf config-manager --set-enabled powertools
dnf -y install libdnet-devel openssl-devel pkgconfig gcc flex bison zlib*
libpcap* pcre* libpcap-devel pcre-devel libnetfilter_queue-devel iptables-devel libdnet* tcpdump epel* wget
```

---

### DAQ 설치

- DAQ는 Data Acquisition Package로 패킷을 가져오는 역할을 합니다.
- Snort가 네트워크 패킷을 읽기 위해 반드시 필요합니다.
```
- dnf install -y daq daq-devel daq-modules libnghttp2
- ln -s /usr/lib64/libdnet.so.1.0.1 /usr/lib64/libdnet.1
```

---

### Snort 설치

- rpm 파일을 다운로드 후 설치합니다.
```
wget https://snort.org/downloads/snort/snort-2.9.20-1.centos.x86_64.rpm
dnf install -y snort-2.9.20-1.centos.x86_64.rpm
```

- 다음 명령어로 설치 여부와 버전을 확인합니다.
```
snort -V
```

![34](/KH_Security/Linux/Firewall/img/34.png)

- `Version 2.9.20 GRE (Build 82)`가 출력되므로 Snort가 정상적으로 설치되었습니다.
- DAQ, PCRE, ZLIB 등의 필요한 라이브러리도 함께 정상적으로 인식되고 있습니다.
- 에러 메시지가 없으므로 실행 환경에도 문제가 없는 상태입니다.

## 기본 환경 설정

### 디렉토리 및 rule 파일 생성

- Snort 실행에 필요한 디렉토리와 rule 파일을 생성합니다.
```
- mkdir /usr/local/lib/snort_dynamicrules
- touch /etc/snort/rules/white_list.rules
- touch /etc/snort/rules/black_list.rules
- touch /etc/snort/rules/local.rules
```

---

### 권한 설정

- 다음 명령어로 Snort 계정이 rule과 로그를 안전하게
  관리할 수 있도록 하기 위해 다음과 같이 설정합니다.
```
- chmod -R 700 /etc/snort
- chmod -R 700 /var/log/snort
- chmod -R 700 /usr/local/lib/snort_dynamicrules
- chown -R snort.snort /etc/snort
- chown -R snort.snort /var/log/snort
- chown -R snort.snort /usr/local/lib/snort_dynamicrules
```

---

### local.rules 수정

- ICMP 탐지를 위한 기본 테스트 rule을 추가합니다.
```
- vi /etc/snort/rules/local.rules
- alert icmp any any -> $HOME_NET any (msg:"ICMP TEST"; sid:1000001; rev:001;)
```

#### 의미

- 외부에서 HOME_NET으로 들어오는 ICMP 패킷을 탐지합니다.
- 탐지되면 ICMP TEST라는 메시지를 출력합니다.

---

### snort.conf 설정 파일 수정

- snort.conf는 Snort의 메인 설정 파일입니다.
- 수정 전 반드시 백업합니다.
```
vi /etc/snort/snort.conf
```

#### snort.conf 수정 전 백업 이유

- snort.conf는 Snort의 핵심 설정 파일입니다.
- 잘못 수정하면 Snort가 실행되지 않거나 탐지가 정상적으로 동작하지 않을 수 있습니다.
- 백업해두면 문제가 생겼을 때 바로 원래 상태로 복구할 수 있습니다.

---

### 주요 설정

- site specific rules 항목에서 `include $RULE_PATH/local.rules` 만 남기고 나머지는 삭제합니다.
- /etc/snort/snort.conf 파일에 다음 설정값 추가합니다.
```
ipvar HOME_NET 192.168.12.0/24
ipvar EXTERNAL_NET !$HOME_NET
var RULE_PATH /etc/snort/rules
var SO_RULE_PATH /etc/snort/so_rules
var PREPROC_RULE_PATH /etc/snort/preproc_rules
var WHITE_LIST_PATH /etc/snort/rules
var BLACK_LIST_PATH /etc/snort/rules
output unified2: filename snort.log, limit 128
```

## 실행 및 테스트

### 콘솔 출력 방식

- 다음 명령어로 탐지 메시지를 화면에 바로 출력합니다.
```
snort -i ens160 -A console -c /etc/snort/snort.conf
```

### alert 파일 생성 방식

- /var/log/snort/alert 파일이 생성됩니다.
```
snort -i ens160 -A fast -c /etc/snort/snort.conf
```

### 로그 확인

- cat /var/log/snort/alert
- snort -r 로그파일

## ICMP 탐지 실습

### AP 시스템 접근 탐지 Rule

- `vi /etc/snort/rules/local.rules` 파일에 다음 Rule을 추가합니다.

![36](/KH_Security/Linux/Firewall/img/36.png)

```
- ICMP 탐지
  - alert icmp any any -> 192.168.12.11 any (msg:"AP ICMP Detect"; sid:1000002; rev:1)
    - 모든 IP에서 192.168.12.11로 들어오는 ICMP(ping) 요청을 탐지합니다.
  - alert icmp $EXTERNAL_NET any -> 192.168.12.11 any (msg:"AP ICMP Detect"; sid:1000003; rev:1)
    - 외부 네트워크(EXTERNAL_NET)에서 192.168.12.11로 들어오는 ICMP 요청만 탐지합니다.
  - alert icmp !$HOME_NET any -> 192.168.12.11 any (msg:"ICMP Detect"; sid:1000004; rev:1)
    - 내부 네트워크(HOME_NET)를 제외한 모든 외부에서 들어오는 ICMP 요청을 탐지합니다.

- SSH 탐지
  - alert tcp any any -> 192.168.12.11 22 (msg:"AP SSH Detect"; sid:1000005; rev:1)
    - 모든 IP에서 192.168.12.11의 22번 포트(SSH)로 접속하는 트래픽을 탐지합니다.

- FTP 탐지
  - alert tcp any any -> 192.168.12.11 21 (msg:"AP FTP Detect"; sid:1000006; rev:1)
    - 모든 IP에서 192.168.12.11의 21번 포트(FTP)로 접속하는 트래픽을 탐지합니다.
```

## ICMP 테스트
```
alert icmp any any -> $HOME_NET any (msg:"ICMP TEST"; sid:1000001; rev:001;)
```

- 모든 곳에서 HOME_NET으로 들어오는 ICMP(ping)를 탐지합니다.
- ping 요청이 들어오면 ICMP TEST 메시지를 출력합니다.

### 실행
- 먼저 IDS 서버에서 `snort -i ens160 -A fast -c /etc/snort/snort.conf` 및  
  `snort -i ens160 -A console -c /etc/snort/snort.conf` 명령어를 입력하여 실행합니다.

- EXTERNAL 서버에서 IDS 서버로 ping을 보냅니다.
```
ping 192.168.12.19
```

![37](/KH_Security/Linux/Firewall/img/38.png)

### 결과 확인 (Console Mode & Fast Mode)

- IDS 서버에서 `snort -i ens160 -A console -c /etc/snort/snort.conf` 입력하여 실행한 뒤 확인합니다.

![38](/KH_Security/Linux/Firewall/img/37.png) 

- `snort -i ens160 -A fast -c /etc/snort/snort.conf`를 입력하여 실행한 뒤 ping을 보냅니다.  
  이후에 `cat /var/log/snort/alert` 입력하여 로그 확인합니다.
![39](/KH_Security/Linux/Firewall/img/39.png)

- 출력 결과를 통해 HOME_NET으로 들어오는 ICMP 요청이 정상적으로 탐지된 것을 확인할 수 있습니다.

## AP ICMP 탐지
```
alert icmp any any -> 192.168.12.11 any (msg:"AP ICMP Detect"; sid:1000002; rev:1)
```

### 결과 확인

- IDS 서버에서 `snort -i ens160 -A fast -c /etc/snort/snort.conf`를 실행합니다.
- EXTERNAL 서버(192.168.11.19)에서 `ping 192.168.12.11`을 수행합니다.
- 이후 `cat /var/log/snort/alert` 명령어로 탐지 로그를 확인합니다.

![40](/KH_Security/Linux/Firewall/img/40.png)

- 출력 결과에서 `AP ICMP Detect` 메시지가 발생한 것을 확인할 수 있습니다.
- 즉, 192.168.12.11로 들어오는 ICMP(ping) 요청이 정상적으로 탐지된 상태입니다.

## AP SSH 탐지
```
alert tcp any any -> 192.168.12.11 22 (msg:"AP SSH Detect"; sid:1000005; rev:1)
```

- 즉, 192.168.12.11의 22번 포트(SSH) 접속이 정상적으로 탐지된 상태입니다.

## 요약 정리

- Snort는 대표적인 IDS로 침입을 탐지하고 로그를 남깁니다.
- DAQ는 패킷 수집 역할을 하며 Snort와 함께 설치됩니다.
- local.rules에 직접 rule을 작성하여 원하는 탐지 정책을 만들 수 있습니다.
- snort.conf에서 HOME_NET과 rule 경로를 정확히 설정해야 합니다.
- console 모드는 화면 출력, fast 모드는 alert 파일 생성 방식입니다.
- ICMP, SSH, FTP, WEB 등 서비스별 탐지 rule을 만들 수 있습니다.
