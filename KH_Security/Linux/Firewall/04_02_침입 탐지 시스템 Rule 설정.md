# 침입 탐지 시스템 기본 Rule 설정

## 개요

- Snort는 대표적인 IDS(Intrusion Detection System)로 네트워크 패킷을 분석하여 침입을 탐지합니다.
- Rule을 통해 어떤 트래픽을 탐지할지 직접 정의할 수 있습니다.
- Snort Rule은 Header와 Option으로 구성되며, 어떤 패킷을 어떻게 탐지할지 설정합니다.

--

## Snort Rule 구조

- Snort Rule은 Header와 Option 두 부분으로 구성됩니다.
- Header는 어떤 패킷을 탐지할지 정하고,
- Option은 탐지 후 어떤 메시지를 남길지, 어떤 조건을 더 볼지 설정합니다.

- `alert icmp any any -> 192.168.12.11 any (msg:"AP ICMP Detect"; sid:1000002; rev:1)`

| 구분 | 항목 | 의미 | 예시 |
|---|---|---|---|
| Header | action | 조건에 맞았을 때 수행할 동작 | alert |
| Header | protocol | 탐지할 프로토콜 지정 | icmp, tcp |
| Header | s.ip | 출발지 IP 주소 (source IP) | any, $HOME_NET |
| Header | s.port | 출발지 포트 번호 (source port) | any, 80 |
| Header | -> | 트래픽 방향 표시 | -> |
| Header | d.ip | 목적지 IP 주소 (destination IP) | 192.168.12.11 |
| Header | d.port | 목적지 포트 번호 (destination port) | any, 22 |
| Option | option | 추가 탐지 조건 및 로그 설정 | msg, sid, rev |

---

### Header
```
alert icmp any any -> 192.168.12.11 any
```

#### action

- 조건에 맞았을 때 어떤 동작을 할지 결정합니다.

- alert : 조건에 맞으면 경고 발생 + 로그 기록
- log : 로그만 기록하고 경고는 발생하지 않음
- pass : 해당 트래픽을 무시
- drop : 패킷 차단 + 로그 기록
- reject : 패킷 차단 + 응답 전송 + 로그 기록
- sdrop : 조용히 패킷 차단 (drop과 유사)

> drop, reject, sdrop은 IPS에서 주로 사용하는 설정입니다.

---

#### protocol

- 어떤 프로토콜을 탐지할지 지정합니다.

- tcp
- udp
- ip
- icmp
- any

---

#### s.ip / d.ip

- 출발지(source IP)와 목적지(destination IP)를 지정합니다.
- host 또는 network 주소를 설정할 수 있습니다.

- 예시
  - any
  - $HOME_NET
  - $EXTERNAL_NET
  - 192.168.10.11
  - 192.168.0.0/24
  - [192.168.10.11,192.168.0.0/24] → 여러 개 지정 (공백 없음)
  - !$HOME_NET
  - !192.168.0.0/24 → 제외

---

#### s.port / d.port

- 출발지(source port)와 목적지(destination port)의 포트 번호를 지정합니다.

- 예시
  - any
  - 80
  - !80
  - :1023
  - 1024:
  - 2048:2056
  - !2048:2056
  - [80,443,2048:2056]

---

### Option
```
(msg:"AP ICMP Detect"; sid:1000002; rev:1)
```

### General

- 일반적인 기본 옵션입니다.
- 가장 자주 사용하는 옵션입니다.

#### 예시
- msg
- sid
- rev

---

### Payload

- 패킷의 내용(message 부분)을 검사하는 옵션입니다.
- 문자열이나 특정 데이터가 포함되어 있는지 확인할 때 사용합니다.

#### 예시
- content
- nocase
- offset
- depth
- distance

---

### Non-payload

- 패킷의 헤더 정보를 검사하는 옵션입니다.
- 내용이 아니라 패킷 구조 자체를 확인합니다.

#### 예시
- fragoffset
- flags
- seq
- ack
- itype

---

### Post-detection

- 탐지 후 어떤 동작을 할지 설정하는 옵션입니다.
- alert 이후 추가 동작을 지정할 때 사용합니다.

---

## General 옵션

### msg

- 경고 메시지를 지정합니다.
- 탐지되었을 때 화면이나 로그에 출력됩니다.

#### 예시
```
msg:"SYNC ATT";
```

---

### sid

- Rule의 고유 식별번호입니다.

#### 규칙
- 100 미만 → 예약됨
- 100 ~ 1,000,000 → Snort 공식 Rule 사용
- 1,000,001 이상 → 사용자 정의 Rule 사용

- 실습에서는 보통 1000001부터 사용합니다.

---

### rev

- Rule 수정 버전을 의미합니다.
- Rule이 수정될 때 번호를 증가시킵니다.

#### 예시
```
rev:1
```
- 꼭 필수는 아니지만 오류 방지를 위해 함께 작성합니다.

---

## 요약 정리

- Snort Rule은 Header와 Option 두 부분으로 구성됩니다.
- Header는 탐지할 대상(프로토콜, IP, 포트 등)을 지정합니다.
- Option은 탐지 조건과 로그 기록 방식(msg, sid, rev 등)을 설정합니다.
- action은 탐지 후 동작을 결정하고, protocol은 어떤 통신을 볼지 지정합니다.
- msg는 경고 메시지, sid는 Rule 식별자, rev는 수정 버전입니다.
- Payload는 패킷 내용, Non-payload는 헤더 정보, Post-detection은 탐지 후 동작을 의미합니다.
